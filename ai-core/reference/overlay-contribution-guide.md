# 科目覆蓋層貢獻指南

> 本文件說明如何為 TeacherOS 的 45 分鐘課堂設計系統新增科目覆蓋層。

---

## 什麼是覆蓋層？

覆蓋層（Subject Overlay）是搭配通用引擎 `subject-lesson-45.md` 使用的科目專屬檔案。引擎定義方法論（工作流、通用原則、節奏模板），覆蓋層定義該科目獨有的設計原則、研究任務、稽核維度與產出規格。

## 貢獻流程

### 1. 草稿撰寫

在你的 workspace 中建立草稿檔案：

```
{workspace}/skills/draft-[科目]-45.md
```

例如：`draft-math-45.md`、`draft-history-45.md`

### 2. 覆蓋層必備結構

```yaml
---
name: [科目]-45
type: subject-overlay
engine: subject-lesson-45
description: [一句話描述]
subjects:
  - [科目代碼]
grades:
  - 7
  - 8
  - 9
---
```

內容區塊（參考 `english-45.md` 的結構）：

- **科目的隱蔽功能**：這個科目在不同年級的發展對應功能
- **科目專屬設計原則**：補充通用引擎的六條原則（不覆蓋）
- **階段一補充**：科目專屬的研究任務
- **階段二補充**：教案格式的額外欄位
- **階段三補充**：科目專屬的稽核維度
- **階段四補充**：產出規格的額外要求

### 3. 測試運行

使用你的草稿覆蓋層搭配通用引擎進行至少一次完整的課堂設計，確認：

- 通用引擎 + 覆蓋層能順利協作
- 科目專屬原則不與通用原則矛盾
- 產出品質符合預期

### 4. 提交審核

將草稿與測試結果交給 David 審核。審核通過後，David 會將覆蓋層正式發佈至：

```
/ai-core/skills/[科目]-45.md
```

## 設計原則

- **精簡**：覆蓋層只寫通用引擎沒有覆蓋到的內容，不重複
- **補充而非覆蓋**：科目原則是「追加」，不是「取代」通用原則
- **規則密度控制**：科目專屬原則建議 3-4 條，稽核維度 2-3 項，避免 AI 執行時的認知過載
