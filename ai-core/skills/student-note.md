# skill: student-note — 學生觀察記錄

記錄特定學生的學習觀察或個案備註，累加至該學生的個人 Markdown 紀錄檔。

## 核心原則

- **一人一檔**：每位學生只有一個紀錄檔，所有紀錄累加在同一檔案中
- **只加不刪**：舊紀錄永不覆蓋、永不刪除
- **不分科目**：所有科目的觀察記錄集中在同一檔案，用 tag 區分
- **日期 + tag**：每筆紀錄以日期為標題，附帶 tag 標記科目與項目類型

## 參數

班級代碼 + 學生識別（座號、代號或名字）。
例：「9C，第 5 號」「9C，小明」「9C，後排那個」

## 根目錄

以 Repo 根目錄為基準（相對路徑）。
AI 自動偵測根目錄位置：嘗試 `git rev-parse --show-toplevel`，或從當前已知的工作目錄推斷。

## 檔案結構

### 存放位置

```
{workspace}/projects/class-{code}/student-notes/{ID}-{姓名}.md
```

例：`workspaces/Working_Member/Codeowner_David/projects/class-9c/student-notes/9C-05-陳彥妤.md`

### 檔案格式

```markdown
---
aliases:
  - "陳彥妤 學習紀錄"
  - "9C-05 陳彥妤"
name: 陳彥妤
id: 9C-05
class: 9C
---

# 陳彥妤（9C-05）學習紀錄

---

## 2026-03-13 #英文 #課堂觀察

（紀錄內容）

---

## 2026-03-20 #導師 #個案討論

（紀錄內容）
```

### Tag 規範

每筆紀錄的 `##` 標題格式：`## YYYY-MM-DD #tag1 #tag2 ...`

常用 tag（教師可自由擴充）：

**科目類**：`#英文` `#主課程` `#導師` `#農場實習` `#服務學習`
**項目類**：`#課堂觀察` `#個案討論` `#親師溝通` `#同儕互動` `#情緒狀態` `#學習表現` `#行為紀錄`
**工作項目類**：`#DI調整` `#評量回饋` `#跟進事項`

## 執行步驟

### Step 1 — 定位學生紀錄檔

1. 從 `acl.yaml` 取得當前使用者的 workspace 路徑
2. 讀取 `{workspace}/projects/class-{code}/roster.yaml`，比對學生識別（座號、名字、代號）
3. 找到對應的紀錄檔：`{workspace}/projects/class-{code}/student-notes/{ID}-{姓名}.md`

若紀錄檔不存在，自動建立（從 roster.yaml 取得 name、id、class，含 notes 欄位則一併帶入 frontmatter）。
frontmatter 必須包含 `aliases` 欄位，格式為 `["{name} 學習紀錄", "{id} {name}"]`，確保 Obsidian 快速搜尋能找到。

若 `student-notes/` 資料夾不存在，先建立資料夾，再建立檔案。

### Step 2 — 收集觀察記錄（語音友善）

提示教師：「請描述你觀察到的情況。用語音說就好，我來整理。」

AI 將教師口述整理為客觀描述，並從內容推斷合適的 tag。整理後顯示給教師確認：

```
日期：2026-03-13
Tag：#英文 #課堂觀察
內容：
（整理後的客觀描述）

確認寫入嗎？要調整 tag 或內容嗎？
```

### Step 3 — 寫入紀錄

確認後，將紀錄**附加至檔案最下方**：

```markdown

---

## 2026-03-13 #英文 #課堂觀察

（紀錄內容）
```

寫入邏輯：
- 讀取現有檔案內容
- 在最後一行之後附加 `---` 分隔線 + 新紀錄
- 不修改任何既有內容

### Step 4 — 後續建議

記錄完成後，依情況主動提示：

- 「還有其他學生要記錄嗎？」（批次記錄模式）
- 若觀察涉及學習困難：「需要對這位學生設計差異化支援嗎？說『查 DI』我來協助。」
- 若觀察涉及情緒或人際問題：「需要進入導師業務流程嗎？說『導師業務』繼續。」

## 萃取與查詢

教師可隨時要求從紀錄中萃取資訊：

- 「幫我看一下 9C-05 所有英文相關的紀錄」→ 搜尋該生檔案中含 `#英文` 的段落
- 「9C 班有哪些學生最近有情緒狀態的觀察」→ 掃描全班 student-notes，搜尋 `#情緒狀態`
- 「整理 9C-05 的個案摘要」→ 讀取該生全部紀錄，產出結構化摘要

## 注意事項

- 觀察記錄採客觀描述（「站起來走動了三次」），不含評判語言（不寫「很不專心」）
- 紀錄檔含真實姓名，應列入 `.gitignore`，不上傳 GitHub
- 語音輸入：「記一下 9C 五號」「幫我記錄陳彥妤」即可觸發
- 若教師一次要記錄多位學生，進入批次模式：逐一記錄，每位確認後自動問「下一位？」
