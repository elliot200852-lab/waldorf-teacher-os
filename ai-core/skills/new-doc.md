# skill: new-doc — 新 Markdown 文件放置確認

每當 AI 需要新建一個 Markdown（.md）文件時，必須先與教師確認存放位置與檔名，避免檔案被放到錯誤的目錄。

## 適用範圍

- **觸發條件**：AI 判斷需要新建 `.md` 文件時自動進入此流程——無論是教師明確要求，或 AI 在工作流中需要產出新文件。
- **不觸發**：YAML 狀態檔（`*.yaml`）、設定檔、或對既有檔案的編輯，不進入此流程。

## 根目錄

以 Repo 根目錄（`git rev-parse --show-toplevel`）為基準。

## 執行步驟

### Step 0 — 判斷脈絡

從當前工作狀態中蒐集資訊：

- 當前載入的 `project.yaml`（哪個專案？哪個班級？）
- 當前的 `working/*.yaml`（哪條工作線？）
- 文件的主題與用途

這些資訊用於 Step 1 的路徑建議。

### Step 1 — 建議存放位置

向教師提出存放路徑建議，格式如下：

> **新文件存放位置確認**
>
> 根據目前的工作脈絡，建議存放在：
> `projects/<專案>/content/<檔名>.md`
>
> 是否需要建立新的子資料夾？
> 完整路徑：`/Users/.../projects/g9/content/<檔名>.md`
>
> 請確認，或告訴我要放在哪裡。

**建議原則：**

- 優先建議 `projects/<當前專案>/content/` 目錄
- 若文件屬於新主題且現有資料夾不適合，主動建議建立新子資料夾並命名
- 若無法判斷專案歸屬，列出 2～3 個候選位置讓教師選擇

教師確認後，進入 Step 2。

### Step 2 — 確認檔名

建議檔名格式為**英文在前、中文在後**：

```
英文描述_中文名稱.md
```

範例：
- `farm-internship-record_農場實習教學紀錄.md`
- `sisyphus-poem_快樂的薛西佛斯.md`
- `parent-letter-march_三月家長信.md`

**命名規則：**

| 部分 | 格式 | 說明 |
|------|------|------|
| 英文前綴 | kebab-case | 簡短描述文件主題，方便 Tab 補全與 CLI 操作 |
| 分隔符 | `_`（底線） | 連接英文與中文部分 |
| 中文後綴 | 自然語言 | 人類可讀的完整名稱 |
| 副檔名 | `.md` | Markdown 格式 |

向教師展示建議檔名，確認或修改後進入 Step 3。

### Step 3 — 寫入檔案

教師確認路徑與檔名後，**立即建立檔案並寫入內容**。

- 若目標資料夾不存在，先建立資料夾
- frontmatter 必須包含 `related: []`（空陣列）。AI 根據當前工作脈絡填入初始連結：
  - 配對檔案（教案 ↔ 學習單）
  - 前後堂次（wiki 備課日誌）
  - 相關概念頁（作者、文本、主題）
  - 若為素材檔且已有 `published_url`，一併填入
  - 連結規範見 `ai-core/reference/wikilink-protocol.yaml`
- 教案類型的檔案（`*-教師教案.md`、`*-教學參考.md`）在 body 末尾加 `## 連結` 區段
- 學習單類型（`*-學習單.md`）只加 frontmatter `related:`，不加 body 連結（學生會看到）
- 寫入完成後，回報：

> 已建立：`projects/g9/content/farm-internship-record_農場實習教學紀錄.md`

### Step 4 — 詢問是否複製到其他位置

檔案建立完成後，詢問教師：

> 是否要將此文件複製到其他資料夾？
> （例如：Personal-Asset-base、其他班級目錄、或其他指定位置）

- 教師回答「是」→ 確認目標路徑後執行複製
- 教師回答「不用」→ 流程結束

## 注意事項

- 此流程在每次新建 .md 文件時都必須執行，不可跳過
- 若教師在指令中已明確指定路徑（如「放到 g9/content」），仍需確認檔名格式，但可省略路徑建議步驟
- 全程使用繁體中文回應
- 此 skill 不處理 Google Drive 上傳，需要上傳時請另行觸發 drive skill
