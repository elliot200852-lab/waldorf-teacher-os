# skill: lesson-evaluator — 課程設計品質評估

## 定位

這是 TeacherOS 的課程品質閘門，與 lesson-engine 形成 **Generator-Evaluator** 配對。

- **lesson-engine**（生成器）：設計課程，內建 Stage 3 稽核員做 10 原則檢查
- **lesson-evaluator**（評估器）：補充 Stage 3 未覆蓋的維度——課型適配、節奏量化、教師操作負荷、應用語境、跨課堂模式

本技能**不重複** Stage 3 的工作。若 audit-log.csv 已有該教案的紀錄，直接引用；若無紀錄，才執行簡化版 10 原則掃描。

### 架構參考

本技能的結構跟隨 `story-verify.md` 的成熟模式：多維度檢查 → 結構化報告 → 三級判定 → 明確後續動作。

## 適用對象

所有教師。

## 根目錄

以 Repo 根目錄為基準。

## Context Isolation 三級制

| 級別 | 觸發方式 | 隔離程度 | 適用場景 |
|------|---------|---------|---------|
| **快速** | 同 session 直接說「評估課程」 | 無隔離 | 日常備課，快速自檢 |
| **標準** | 新 session 說「評估 week7-3-4」 | 自然隔離（新對話無設計記憶） | 重要教案、Block 結束時 |
| **深度** | 用 Agent tool 開 subagent 跑本技能 | 強制隔離 | 學季大綱回顧、跨 Block 審計 |

快速模式下，AI 須意識到「我剛設計了這堂課」的偏見，刻意從批判角度審視。
標準模式是推薦的日常用法——備課完存檔，下次開新 session 再跑評估。

## 執行步驟

### Step 0 — 課型辨識（所有後續步驟依賴此判定）

**在任何評估之前，必須先判定課型。** 不同課型有不同的節奏標準和 DI 期待。

從教案的標題、文法教學模式、活動結構判斷課型：

| 課型 | 辨識線索 | 節奏特徵 | DI 層次期待 |
|------|---------|---------|------------|
| **教學課**（一般） | 有新素材引入 + 練習 + 延伸 | 吸呼交替頻繁，標準吸氣比 30-45% | 依活動需求，通常 2-3 層 |
| **測驗課** | 有「正式測驗」「驗收」「考試」段落 | 測驗段允許長吸氣，重點看前後的設計 | 測驗本身分版即可，不需多層延伸 |
| **概念引入課** | 第一次建立新文法框架或概念系統 | 教師帶的比例天然偏高，但**必須穿插應用** | **兩層即可**（鷹架版 + 開放版），不需 ABCD 四層 |
| **複習課** | 整合已學過的內容 | 學生產出為主，吸氣比應低 | 可多層，因為學生已有基礎 |
| **連堂課**（90min） | 時長 90 分鐘 | 必須有明確的呼吸轉換段 | 可分前後半段分別設計 |
| **閱讀課** | 以文本閱讀為核心 | 朗讀+個人閱讀是長吸氣，需搭配討論/書寫呼氣 | 依文本難度分層 |
| **閱讀課（教學參考模式）** | 教學參考指南+學習單，無逐分鐘教案 | 同閱讀課，但節奏分析從活動建議估算 | 同閱讀課 |

**閱讀課教學參考模式（校準後新增）：**

> 校準來源：week6-3-4 教師評分 4/5，evaluator 原判 REVISE。教師確認：一份好的教學參考指南+好的學習單，教師可以獨立操作，不需要逐分鐘教案。

並非所有課都需要完整教師教案。閱讀課（或其他以文本/素材為核心的課）可以用「教學參考模式」備課：

| 組件 | 必要性 | 說明 |
|------|--------|------|
| 教學參考指南 | **必要** | 含：篇章概述、核心主題、語言特點、ABCD 差異化教學建議、活動建議、與課程大綱的銜接 |
| 學習單（分層） | **必要** | 含：具體任務、認知梯度清晰的 D/C/B/A 層次 |
| 教師版答案 | **必要** | 含：參考答案、批改提示、批改時間估算 |
| 逐分鐘教案 | **非必要** | 教師根據參考指南自行安排節奏 |

**當課型為「閱讀課教學參考模式」時：**
- Step 4 節奏分析：從活動建議估算，標記「估算值」，不因缺逐分鐘計畫而判 flag
- Step 5 教師操作負荷：正常評估（教學參考模式不影響操作負荷判定）
- 不因缺正式教案而自動判 REVISE

**判定「教學參考模式」是否合格的標準：**
- 教學參考指南是否涵蓋足夠資訊讓教師獨立備課？
- 學習單品質是否達標（認知梯度、內容結合度、應用層次）？
- 整體素材是否構成完整的教學支持系統？

**課型判定直接影響後續所有維度的合格標準。** 用教學課的標準去評測驗課，會誤判。

### Step 1 — 定位教案

1. 若教師指定了週次或檔名 → 從 `content/` 和 `reference/` 資料夾找到對應檔案
2. 若未指定 → 讀取 session.yaml 的 `output_files.units_completed`，取 `last_updated` 最近的一份
3. 確認備課素材存在——接受兩種格式：
   - **標準格式**：教師教案（content/）+ 學習單（content/）
   - **教學參考模式**：教學參考摘要（content/）+ 學習單（content/）+ 教師版（content/）
4. 若兩種格式都找不到，回報：「找不到可評估的教案。請指定週次或先跑 lesson-engine。」
5. 根據找到的格式，在 Step 0 課型辨識中標記為「標準教案」或「教學參考模式」

### Step 2 — 維度 1：Stage 3 稽核結果（引用，非重做）

讀取 `ai-core/reference/audit-log.csv`，搜尋匹配的 `lesson_id`。

| 情境 | 處理 |
|------|------|
| audit-log 有對應紀錄 | 直接引用 p01-p10 結果，flag 項目列入報告 |
| audit-log 無紀錄（非 lesson-engine 產出） | 執行簡化版 10 原則掃描，逐項標記 pass/flag |

簡化版 10 原則掃描（僅在無 audit-log 紀錄時使用）：

| 原則 | 檢查問題 | 判定 |
|------|---------|------|
| P01 生成性負載 | 是否有至少一個環節要求學生獨立思考？ | pass/flag |
| P02 差異化 | 每個主要活動是否有適當層次的參與方式？（注意：不是越多層越好，見 Step 3b） | pass/flag |
| P03 學習路徑 | 頭心手 + 至少兩種學習優勢？**注意：學習單和課堂流程分開評估**（見下方校準註記） | pass/flag |
| P04 年級發展 | 素材與活動是否符合年級心理需求？ | pass/flag |
| P05 口頭書寫 | 比例是否刻意設計？ | pass/flag |
| P06 Follow-up | 是否有延伸體驗？ | pass/flag |
| P07 圖像優先 | 新概念是否先圖像後概念？ | pass/flag |
| P08 意志是做 | 意志環節是學生做的嗎？ | pass/flag |
| P09 整體到部分 | 新素材先整體後局部？**文法教學必須是歸納法**（見下方校準註記） | pass/flag |
| P10 節律重複 | 重要概念有螺旋回歸設計？ | pass/flag |

**P03 校準註記——學習單與課堂流程分開評估（week7-1 校準）：**

> 校準來源：week7-1 教師評分 3/5。課堂活動有卡片分類（動覺）和海報（圖像），但學習單本身全是文字。evaluator 把課堂活動的動覺算進了學習單的評估，導致誤判。

P03 的檢查必須分兩個層次：
- **課堂流程**：整堂課的頭心手和學習優勢覆蓋
- **學習單本身**：學習單作為獨立文件，是否只有單一學習入口（純文字）？

若學習單全為文字型（閱讀+書寫），即使課堂流程有動覺/圖像活動，學習單本身仍應標記 flag：「學習單缺動作/圖像入口」。

**P09 校準註記——文法教學中「整體到部分」= 歸納法（week7-1 校準）：**

> 校準來源：week7-1 教師評分 3/5。教師指出「先給全部 connector 框架再逐一講解」是演繹法（deductive），不是華德福的「從整體到部分」。

GA 294 的「從整��到部分」在**文法教學**中的正確實踐是**歸納法**（inductive）：

| 方法 | 流程 | 華德福判定 |
|------|------|----------|
| **歸納法**（正確） | 學生在語境中遇見語言現象 → 自己注意到規律 → 教師命名並系統化 | PASS |
| **演繹法**（錯誤） | 教師先呈現完整分類框架 → 再逐一填入成員/示範 | FLAG |

「先給所有連接詞的全貌表，再一個一個細講」= 演繹法。
「先讓學生在 Mango Street 文本中遇到連接詞 → 自己發現它們的功能 ��� 教師歸類」= 歸納法。

注意：這不是否定「整體到部分」原則——而是在文法教學脈絡中，「整體」不是指「全部規則的概覽」，而是指「完整的語言經驗」（例如一整篇故事），「部分」是從中提取的語言規律。

### Step 3 — 維度 2：DI 適配檢查（校準後大幅修正）

**本維度的核心原則：差異化的層次必須適配課型，不是越多越好。**

> 校準來源：week6-2 教師評分 2.5/5，原因是概念引入課被設計成 ABCD 四層，過度複雜。
> 教師原話：「概念引入分兩個層次就好。重點是後面的活動，要可以讓它應用、有語境。」

**3a. DI 層次適配檢查（新增，最重要）**

根據 Step 0 辨識的課型，判斷 DI 層次是否適當：

| 課型 | 適當的 DI 層次 | 過度差異化 flag |
|------|--------------|---------------|
| **概念引入課** | **兩層**：鷹架版（引導、填空、配對）+ 開放版（造句、應用、延伸） | ABCD 四層 = flag |
| **教學課（一般）** | 2-3 層，依活動複雜度而定 | 超過 3 層且教師操作負荷過高 = flag |
| **測驗課** | 測驗分版（基礎/進階）即可，延伸任務 1-2 層 | 延伸任務超過 2 層 = 不必要 |
| **複習課** | 可多層（學生已有基礎，能自主選擇） | 通常不會過度 |
| **連堂課** | 前半段可少層（全班共同），後半段可多層（深度工作） | 前半段就四層 = flag |

**過度差異化的判定標準：**
- 教師需要同時管理 4 種以上的不同任務 → flag
- 分層增加了教師操作負荷但未增加學習深度 → flag
- 學生被切得太碎，每組只有 2-3 人，教師無法有效巡迴 → flag

**3b. 抽象標籤掃描**

逐行掃描教案文字，偵測以下禁止用語（來自 strategy-output-quality-standard.md）：

| 禁止標籤 | 應替換為 |
|----------|---------|
| 提供鷹架支持 | 描述具體的材料、步驟、教師話語 |
| 延伸挑戰 | 說明挑戰的具體內容與形式 |
| 連結個人興趣 | 描述課堂中連結發生的具體時刻 |
| 提升意義感 | 說明如何讓學生感受到意義 |
| 降低焦慮 | 描述具體的環境設計或教師行為 |

**3c. 策略-內容結合度**

檢查：學習優勢入口和差異化策略是否與**這堂課的具體內容**結合？還是可以原封不動套用到任何一堂課？

判定標準：如果把課程主題換成完全不同的主題，策略描述依然成立 → 判定為「罐頭策略」，標記 flag。

### Step 4 — 維度 3：節奏量化分析（依課型調整標準）

**4a. 解析時間分配表**

從教案中提取每個環節的：
- 名稱
- 時長（分鐘）
- 性質標記：
  - ▲ 吸氣（接收）：教師講述、朗讀、示範、全班聆聽、**安靜測驗**
  - ▽ 呼氣（行動）：學生書寫、討論、分組活動、身體活動、創作、**pair work**
  - ─ 轉換：過渡、移動、歌唱暖身、短暫休息

**關鍵判定原則——互動即動覺：**

> 校準來源：week5-1 教師指出「分組本身就是動覺」；week6-2 教師指出「pair work 自然就動起來了」。

以下活動在計算動覺佔比時，**算作動覺**（即使學生沒有離開座位）：
- Pair work（兩人互動，自然產生轉身、指向、身體語言）
- 小組討論（3-4 人，需要移動座位或轉向）
- 分組活動（需要重新分組、移動位置）
- 上台操作（板書、圈詞、展示）

以下活動**不算動覺**（即使名稱聽起來像）：
- 全班齊讀（坐著朗讀，無互動）
- 個人書寫（安靜獨立作業）
- 教師帶讀後學生跟讀（本質仍是接收）

**4b. 依課型計算節奏指標**

**教學課（一般）標準：**

| 指標 | 合格標準 |
|------|---------|
| 吸氣比 | 30-45% |
| 呼氣比 | 45-60% |
| 轉換比 | 5-15%（0% = FLAG，這是明確缺陷） |
| 最長連續吸氣 | ≤ 15 分鐘 |
| 動覺佔比（含互動式活動） | ≥ 15% |
| 交替頻率 | 45min ≥ 3 次，90min ≥ 5 次 |

**測驗課標準（獨立標準）：**

> 校準來源：week6-1 教師評分 4/5，確認測驗課不能用一般標準。

| 指標 | 合格標準 |
|------|---------|
| 測驗佔比 | ≤ 50%（測驗是核心，允許長吸氣） |
| 暖身+收尾佔比 | ≥ 15%（測驗前後要有節奏設計） |
| 提前完成者銜接 | 有具體的銜接活動（不是「坐著等」） |

測驗課不計算吸呼比和交替頻率——這些指標對測驗課無意義。

**閱讀課標準（含教學參考模式）：**

> 校準來源：week6-3-4 教師評分 4/5，確認閱讀課有自己的節奏邏輯。

| 指標 | 合格標準 |
|------|---------|
| 教師朗讀/導讀段 | 允許長吸氣（≤ 20min），閱讀課的核心是文本體驗 |
| 討論/回應佔比 | ≥ 30%（朗讀後必須有消化空間） |
| 書寫任務佔比 | ≥ 20%（學習單工作時間） |
| 動覺佔比 | ≥ 10%（Connector Hunt、pair work 等互動式活動即可） |

閱讀課的節奏天然是「長吸氣→長呼氣」，不像教學課需要頻繁交替。
教學參考模式下，節奏指標從活動建議估算，標記「估算值」，不因缺逐分鐘計畫而判 flag。

**概念引入課標準：**

> 校準來源：week6-2 教師評分 2.5/5，確認概念引入課教師帶比例天然偏高但必須穿插應用。

| 指標 | 合格標準 |
|------|---------|
| 教師帶入比 | ≤ 50%（允許比一般課高，但不能全堂講） |
| **應用環節佔比** | **≥ 30%**（最重要——概念引入後必須有語境化的應用） |
| 應用環節有 pair/group | **必須**（不是個人填空，是有互動的應用） |
| 轉換比 | ≥ 5%（概念段之間需要微轉換） |

### Step 5 — 維度 4：教師操作負荷評估（校準後新增）

> 校準來源：week5-1 教師指出「Y 字叉路單人操作困難」——單人教師同時貼膠帶、引導動線、觀察反應，負荷偏高。

**本維度檢查教案在實際教室中，單一教師是否能流暢操作。**

**5a. 同時操作任務數**

掃描每個環節，計算教師需要同時處理的任務數：

| 同時任務數 | 判定 |
|-----------|------|
| 1-2 件 | PASS |
| 3 件 | WARN — 「操作緊湊，建議簡化或分步」 |
| 4+ 件 | FLAG — 「單人教師難以同時處理」 |

範例：
- 「發學習單 + 說明規則」= 2 件 → PASS
- 「貼膠帶 + 引導動線 + 觀察學生反應」= 3 件 → WARN
- 「分發兩版測驗卷 + 確認分組 + 計時 + 巡視」= 4 件 → 但可以分步做 → PASS（分步即可降壓）

**5b. 分組巡迴可行性**

若教案有分組/分層活動：

| 分組情境 | 判定 |
|---------|------|
| 2 組（入門+挑戰），教師主要陪伴入門層 | PASS |
| 3 組，教師有明確巡迴策略（先 D → B → A/C） | PASS |
| 4+ 組同時進行不同任務，無巡迴優先序 | FLAG — 過度分流 |

**5c. 課前準備負荷**

| 準備項目數 | 判定 |
|-----------|------|
| ≤ 4 項 | PASS |
| 5-6 項 | WARN |
| 7+ 項 | FLAG — 「準備過重，考慮簡化」|

### Step 6 — 維度 5：應用語境檢查（校準後新增）

> 校準來源：week6-2 教師指出「填連接詞 ≠ 用連接詞」。重點是活動要有語境、有應用、有驗收。

**本維度檢查練習環節是否有意義的應用，而非機械式填空。**

**6a. 應用層次判定**

掃描教案中所有學生練習環節，判定其應用層次：

| 層次 | 定義 | 範例 | 判定 |
|------|------|------|------|
| **機械練習** | 選擇題、配對題、填空（答案唯一） | 「填入正確的連接詞」 | 最低層——單獨出現 = WARN |
| **引導應用** | 有語境的填空或改寫（有上下文） | 「讀完對話後，選填連接詞」 | 中層——OK 但不應是唯一形式 |
| **創造性應用** | 學生在真實語境中使用目標語言 | 「用連接詞描述昨天發生的事」「pair work 互相造句」 | 最高層——必須出現 |
| **社交性應用** | 有互動的應用（pair/group） | 「兩人用片語完成一段對話」 | 加分——自然產生動覺 |

**合格標準：**
- 教案中至少有一個**創造性應用**或**社交性應用**環節
- 如果全部練習都是機械練習 → FLAG
- 如果只有機械練習 + 引導應用，缺創造/社交 → WARN

**6b. 任務自選可行性**（校準後降權）

> 校準來源：week6-1 教師指出「任務自選機制運行不順——學生主動性不高」。

教案中若有「學生自選任務」的設計：
- 不自動列為亮點（之前 evaluator 錯誤地將此列為亮點）
- 標記為 WARN：「自選機制對主動性低的班級可能不適用。備案：教師根據觀察分派。」

### Step 7 — 維度 6：跨課堂單調性偵測

讀取 session.yaml 的 `units_completed`，取最近 3-5 堂（若不足 3 堂則跳過此維度）。

比對以下模式：

| 比對項目 | 警告條件 |
|---------|---------|
| 開場活動類型 | 連續 3 堂使用同一種暖身 |
| 核心活動結構 | 連續 3 堂的主體活動結構雷同 |
| 差異化任務類型 | 連續 3 堂的差異化模式固定化（例：每次都是「A 寫長文、D 填空」） |
| 學習優勢覆蓋 | 近 5 堂中某種學習優勢從未出現 |
| 互動形式 | 近 3 堂都沒有 pair work 或分組 |

有紀錄不足 3 堂時，輸出「跨課堂資料不足，跳過單調性偵測」。

### Step 8 — 維度 7：整體直覺判斷

以下面的提示自問，並誠實回答：

> 如果你是這個班的一個 14 歲學生（或對應年級的學生），上完這堂課你會記住什麼？
> 如果你能說出一個具體的畫面或感受 → PASS
> 如果只能說出抽象的「學到了某個概念」 → WARN
> 如果什麼都說不出來 → FLAG

將回答寫入報告的「整體印象」欄位。

### Step 9 — 綜合判定與產出報告

**判定邏輯：**

| 總判定 | 條件 |
|--------|------|
| **PASS** | 無 flag，所有維度合格（依課型標準） |
| **REVISE** | 有 1-3 個 flag 或 warn，但整體結構完整 |
| **REDO** | 有 4+ 個 flag，或過度差異化 + 缺應用語境（雙殺），或整體判斷為「什麼都記不住」 |

**報告格式：**

```markdown
# 課程評估報告：[班級]-[週次]-[序號]

**評估日期**：[日期]
**教案**：[檔案路徑]
**課型**：[教學課/測驗課/概念引入課/複習課/連堂課/閱讀課]
**總判定**：PASS / REVISE / REDO

---

## 一、節奏分析（[課型]標準）

時間線：[節奏圖]
[依課型列出對應指標和結果]

## 二、10 原則稽核（引用 Stage 3 / 簡化掃描）

[p01-p10 結果摘要，只列 flag 項目]

## 三、DI 適配

課型適配：[PASS / 過度差異化 flag]
雙軸覆蓋：[PASS / 缺項]
抽象標籤：[發現 N 處 / 無]
策略-內容結合度：[PASS / 罐頭策略警告]

## 四、教師操作負荷

同時任務數：[PASS / WARN / FLAG]
分組巡迴：[PASS / FLAG]
課前準備：[PASS / WARN / FLAG]

## 五、應用語境

應用層次：[機械/引導/創造/社交]
有無創造性或社交性應用：[有 / 缺]

## 六、跨課堂模式

[比對結果，或「資料不足」]

## 七、整體印象

學生會記住：[一句話描述]

---

## 修正建議

[若判定為 REVISE 或 REDO，列出具體修正建議，每條包含：]
1. **問題**：[具體問題]
   **建議**：[具體修改方向]
   **位置**：[教案中的哪個環節]

## 亮點

[列出 1-3 個真正值得肯定的設計——不是常規操作]
```

### Step 10 — 寫入評估紀錄

1. 報告存入 `{workspace}/projects/class-{code}/{subject}/reviews/[週次]-evaluation.md`
   - 若 `reviews/` 資料夾不存在，建立之
2. 更新 session.yaml，在 `evaluation_log` 追加一筆（append，不覆蓋）：

```yaml
evaluation_log:
  - unit: [週次]
    date: [今日日期]
    verdict: [PASS/REVISE/REDO]
    lesson_type: [課型]
    rhythm: [吸氣比:呼氣比:轉換比]（或測驗課型專用指標）
    flags: [flag 項目列表]
    strengths: [亮點列表]
```

若 session.yaml 中尚無 `evaluation_log` 區塊，新增之。

## 後續動作

| 判定 | 後續 |
|------|------|
| **PASS** | 「評估通過。要美化輸出 PDF 嗎？」 |
| **REVISE** | 「有 [N] 個需修改的地方。要現在處理嗎？還是記下來下次處理？」 |
| **REDO** | 「建議重新設計。主要問題是：[一句話摘要]。要開始嗎？」 |

## 亮點判定標準（校準後收緊）

> 校準來源：week6-1 教師指出「不當場對答案是國中基本操作，不是亮點」。

亮點必須是**超出常規操作的設計巧思**。以下**不算亮點**：
- 國中教學的基本操作（對答案、收學習單、計時等）
- 任務自選機制（在主動性低的班級效果不佳）
- 標準的 DI 分層（有 A/B/C/D 不代表做得好）

以下**算亮點**：
- 跨週延續的視覺鷹架（如 Connector Family 樹）
- 螺旋式語言複習設計（同一結構在不同認知層次循環）
- 保護學生尊嚴的操作細節（如測驗分版外觀一致）
- 身體先行的概念引入（如 Y 字叉路）
- 讓素材「以配角身分不知不覺出現」的節律重複

## 注意事項

- 全程使用繁體中文
- **課型辨識是第一步**——所有後續判定都依賴課型。用錯標準會系統性誤判。
- 評估報告以「缺什麼」為主，不重複列出已通過的項目
- 但**亮點**必須列出——evaluator 不只是挑錯機器
- 跨課堂偵測需要累積資料，前 3 堂課的評估報告中此維度會顯示「資料不足」
- 節奏量化的合格標準是**參考值**，不是硬性門檻——教師可根據實際教學經驗調整
- 校準期（前 5 次使用）：evaluator 跑完後主動問教師：「這個判定跟你的直覺一致嗎？」
- 班級特性會影響標準——讀取 class project.yaml 了解班級動力，必要時放寬或收緊
- 此技能適用於所有 AI 平台（Claude Code、Gemini、ChatGPT 等），無平台依賴
