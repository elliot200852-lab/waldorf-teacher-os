# skill: teaching-log — 教師教學紀錄

記錄教師的教學回顧、課堂觀察或工作備忘，累加至教師個人的 `teaching-log.md`。

## 核心原則

- **一師一檔**：每位教師只有一個 `teaching-log.md`，所有紀錄累加在同一檔案中
- **只加不刪**：舊紀錄永不覆蓋、永不刪除
- **跨班跨科**：所有班級、所有科目的教學紀錄集中在同一檔案，用 tag 區分
- **日期 + tag**：每筆紀錄以日期為標題，附帶 tag 標記班級、科目與項目類型

## 參數

選填。教師可直接說「教學紀錄」開始，AI 從口述內容推斷班級與科目。
也可指定：「教學紀錄 9C 英文」。

## 根目錄

以 Repo 根目錄為基準（相對路徑）。
AI 自動偵測根目錄位置：嘗試 `git rev-parse --show-toplevel`，或從當前已知的工作目錄推斷。

## 檔案結構

### 存放位置

```
{workspace}/teaching-log.md
```

例：`workspaces/Working_Member/Codeowner_David/teaching-log.md`
例：`workspaces/Working_Member/Teacher_林詩閔/teaching-log.md`

### 檔案格式

```markdown
---
name: 林信宏（David）
role: Codeowner
classes: [9C]
---

# 林信宏（David）教學紀錄

---

## 2026-03-13 #英文 #課堂觀察 #9C

今天第三節上 Unit 4 閱讀，發現 B 類學生在分組討論時參與度明顯提高...

---

## 2026-03-13 #導師 #親師溝通 #9C

跟 9C-08 的家長通話，討論近期在校情緒波動...

---

## 2026-03-15 #農場實習 #教學流程 #9C

返校週分享會的流程需要調整，學生上台時間太短...
```

### Tag 規範

每筆紀錄的 `##` 標題格式：`## YYYY-MM-DD #tag1 #tag2 ...`

常用 tag（教師可自由擴充）：

**班級類**：`#9C` `#9D` `#8A` `#7A`（依實際班級）
**科目類**：`#英文` `#主課程` `#歷史` `#導師` `#農場實習` `#服務學習`
**項目類**：`#課堂觀察` `#教學流程` `#教學反思` `#親師溝通` `#班級經營` `#評量設計` `#差異化` `#備課筆記`
**工作項目類**：`#區塊結束` `#學季規劃` `#跨科整合` `#行政事務`

## 執行步驟

### Step 1 — 定位教師紀錄檔

1. 從 `acl.yaml` 取得當前使用者的 workspace 路徑
2. 讀取 `{workspace}/teaching-log.md`

若紀錄檔不存在，自動建立：
- 從 `{workspace}/teacheros-personal.yaml` 取得教師姓名與角色
- 建立 frontmatter（name、role、classes）
- 建立標題

### Step 2 — 收集教學紀錄（語音友善）

提示教師：「請說說你想記錄的內容。用語音說就好，我來整理。」

AI 將教師口述整理為清晰的段落，並從內容推斷合適的 tag（班級、科目、項目類型）。整理後顯示給教師確認：

```
日期：2026-03-13
Tag：#英文 #課堂觀察 #9C
內容：
（整理後的內容）

確認寫入嗎？要調整 tag 或內容嗎？
```

### Step 3 — 寫入紀錄

確認後，將紀錄**附加至檔案最下方**：

```markdown

---

## 2026-03-13 #英文 #課堂觀察 #9C

（紀錄內容）
```

寫入邏輯：
- 讀取現有檔案內容
- 在最後一行之後附加 `---` 分隔線 + 新紀錄
- 不修改任何既有內容

### Step 4 — 後續建議

記錄完成後，依情況主動提示：

- 「還有其他要記錄的嗎？」
- 若內容涉及特定學生：「要同步記錄到該學生的個人紀錄嗎？說『記錄學生 [班級] [學生]』。」
- 若內容涉及區塊結束反思：「這是區塊結束的反思嗎？要觸發完整的 block-end 流程嗎？」
- 若同堂課有備課反思（`wiki/{class}-{subject}-w{week}-*-reflect.md`）：「這堂課有備課反思紀錄，要連結進來嗎？說『合併之前的反思』。」

**合併備課反思觸發詞：** 教師在 teaching-log 過程中說「合併之前的反思」「連結 wiki reflect」「加上備課反思」「把反思連進來」時，AI 執行：
1. 從當前紀錄推斷班級 + 科目 + 近期週次
2. 掃描 `wiki/` 找到匹配的 `-reflect.md` 檔
3. 在本筆紀錄末尾加上：`> 備課反思：[[{class}-{subject}-w{week}-{seq}-reflect]]`
4. 反向更新 wiki-reflect 檔的 `related:`，加上 `[[teaching-log#{YYYY-MM-DD heading}]]`

## 萃取與查詢

教師可隨時要求從紀錄中萃取資訊：

- 「幫我看一下最近所有英文課的教學紀錄」→ 搜尋含 `#英文` 的段落
- 「這學期 9C 的導師紀錄整理一下」→ 搜尋含 `#導師 #9C` 的段落
- 「整理我上個月的教學反思」→ 依日期篩選，產出摘要

## 與其他技能的關係

- **student-note**：記錄學生個人觀察（主體是學生）。teaching-log 記錄教師視角的教學觀察（主體是教師）。兩者可互相參照但不重複。
- **block-end**：區塊結束時的結構化反思，有固定的問答流程。teaching-log 更自由，日常隨時可記。block-end 的反思內容可以同步寫入 teaching-log。
- **wrap-up**：收工時同步進度。teaching-log 是教學內容的累積紀錄，不會在 wrap-up 時自動觸發。
- **wiki-reflect**：備課過程的設計反思（主體是設計決策，存入共享 `wiki/`）。teaching-log 記錄上課後的教學觀察（主體是課堂經驗，存入個人 workspace）。一個看「為什麼這樣設計」，一個看「上完課發生了什麼」。教師做 teaching-log 時可說「合併之前的反思」，AI 自動連結同堂課的 wiki-reflect。

## 注意事項

- 語氣自由，可以包含直覺觀察、情緒感受、未成形的想法
- 紀錄不需要結構化——教師怎麼說就怎麼記，AI 只做最低限度的整理
- 語音輸入：「今天上課的心得記一下」「教學紀錄」即可觸發
- 此技能為所有教師通用，任何已註冊的教師都可使用
