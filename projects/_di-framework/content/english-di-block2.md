# 英文課 DI 設計模板｜區塊二：班級實際教學

> **所屬模板：** `_di-framework/content/english-di-template.md`（主控索引）
> **前置條件：** 建議先完成區塊一產出的 `english-syllabus.md`，作為基準參照
> **品質標準：** 所有策略分析產出必須符合 `_di-framework/content/strategy-output-quality-standard.md`
> **品質範例：** `_di-framework/reference/strategy-analysis-quality-example.md`

---

## 本區塊包含兩個子區塊

| 子區塊 | 名稱 | 核心功能 |
|--------|------|---------|
| 2-A | 單元教學流程設計 | 教學單元規劃 + 逐節 45 分鐘課堂流程 |
| 2-B | 差異化任務與驗收回饋機制設計 | 課中差異化延伸 + 家庭任務 + 減負驗收機制 |

2-A 與 2-B **可以同步推進**，不強制先後順序。
但建議在 2-A 的單元策略表完成後，再進入 2-B 的任務設計，確保任務與課堂流程對齊。

---

## 互動模式：混合式

本區塊採**混合式互動**：

1. AI 主動提問，收集教學方向與關鍵資訊
2. 教師可隨時提供文件素材（文本、講義、圖片等），AI 將其納入設計
3. AI 產出策略分析表供教師選擇，教師決策後 AI 產出完整文件

# 2-A：單元教學流程設計

## 設計邏輯：兩層結構

```
第一層：教學單元（彈性，教師決定）
  │
  │  三種可能類型：
  │  ├─ 單節課型：1 節 45 分鐘，獨立主題
  │  ├─ 多節課型：4 或 6 節課為一個單元
  │  └─ 主題式  ：一個主題跨越不定節數
  │
  ↓  不論哪一種，最終都分解為：
  
第二層：每一節課（固定結構，45 分鐘）
  ┌────────────────────────────────┐
  │ 1. Warm-up（暖身）              │
  │ 2. Main Activity（課程主要活動） │
  │ 3. Check / Assessment（驗收評量）│
  │ 4. Closing（分享或回顧）         │
  └────────────────────────────────┘

每一節課都是獨立完整的教學單位。
```

---

## Phase 1 — 策略層（AI 主導分析）

### Step 1：收集教師輸入

AI 向教師確認以下資訊：

| 輸入項目 | 說明 | 必要性 |
|----------|------|--------|
| 單元主題 | 這個單元要教什麼（例如：The Giver、描述文寫作、英文歌曲分析） | 必要 |
| 單元類型 | 單節課 / 多節課（幾節？）/ 主題式（預計跨幾節？） | 必要 |
| 教學目標 | 學生在這個單元結束後要能做到什麼 | 必要 |
| 核心文本或素材 | 教師提供的閱讀材料、影片、歌曲、圖片等 | 有就提供 |
| 教師的初步想法 | 想嘗試的教學方法、直覺靈感、注意事項 | 有就提供 |

> 教師可以用語音、文字或直接提供檔案。
> AI 應主動整理教師的口述內容，確認理解正確後再進入 Step 2。

### Step 2：AI 讀取系統設定

AI 必須在分析前讀取以下檔案（若尚未在本次對話中載入）：

```
1. ai-core/teacheros.yaml（教師身份與教學哲學）
2. projects/_di-framework/project.yaml（DI 框架規則）
3. projects/_di-framework/content/strategy-output-quality-standard.md（品質標準）
4. projects/class-{班級}/working/students.yaml（學生 DI 樣態）
5. projects/class-{班級}/working/english-session.yaml（進度錨點）
```

### Step 3：產出策略分析表

AI 產出一份完整的策略分析表，包含兩個軸線的建議。

**產出前必須讀取品質標準**（`strategy-output-quality-standard.md`），
確保達到以下基準後才呈現給教師。

#### ABCD 差異化策略建議（必須達到的品質標準）

每一類學生（A / B / C / D）的建議必須：

1. **以心理素描開頭**：描述這類學生的內在狀態與需求
2. **策略精神副標題**：例如「給他們值得掙扎的問題」「先讓他們覺得安全」
3. **具體課堂畫面**：每一條策略都描述老師在教室中可以做的動作、話語、材料
4. **禁止抽象標籤**：不可用「鷹架支持」「延伸挑戰」「連結興趣」作為策略本身
5. **每類至少 3 條**：可以多但不可少

#### 學習優勢入口建議（必須達到的品質標準）

每一種優勢（語言型 / 圖像型 / 動作型 / 藝術型）必須：

1. **與本單元具體內容結合**：不是泛用型建議
2. **完整活動描述**：做什麼、怎麼做、為什麼能觸及這類學生
3. **可直接操作**：老師讀完後能想像自己在課堂上做這件事

#### 品質自檢清單（AI 產出後必須核對）

- [ ] 每類學生是否以心理素描開頭？
- [ ] 每條策略是否描述了課堂中實際會發生的畫面？
- [ ] 是否消除了所有抽象標籤？
- [ ] 學習優勢建議是否跟本單元的具體內容結合？
- [ ] 整體語氣是否像教師之間的專業對話？
- [ ] 老師讀完後能否直接拿走用？

---

## Phase 2 — 教師選擇層（教師主導決策）

### AI 呈現策略表後，教師逐節決定：

| 決策項目 | 說明 |
|----------|------|
| 這節課的重點策略 | 從策略表中選擇：這節課著重在哪些學生特質？ |
| 學習優勢入口 | 這節課主要走哪 2-3 種優勢入口？ |
| 差異化重點 | 這節課主要關照 ABCD 中的哪幾類？如何分層？ |
| 補充想法 | 教師可以修改、替換、或新增 AI 未想到的做法 |

> 教師不需要對每一節課都做完整選擇。
> 可以對第一節課細選，後續幾節課只說「延續同方向」或「換個重點到 B/D」。
> AI 應根據教師的選擇自動調配後續節次的平衡。

---

## Phase 3 — 產出層（AI 執行產出）

### 每一節課的產出格式

AI 根據教師的選擇，為每一節 45 分鐘產出完整教學流程：

```markdown
# {單元名稱}｜Session {n}（{日期 / 第幾節}）

## 本節概覽
- 核心目標：{本節要達成什麼}
- DI 重點：{本節著重 A/B/C/D 中的哪幾類}
- 學習優勢入口：{本節使用的優勢類型}

---

## 1. Warm-up（暖身）｜約 {n} 分鐘

{具體的暖身活動描述。包含：做什麼、怎麼做、教師說什麼。}

---

## 2. Main Activity（課程主要活動）｜約 {n} 分鐘

### 全班共同活動
{核心教學活動的完整描述。}

### 差異化安排
{此處依 Phase 2 教師選擇的差異化重點展開。
  具體說明不同學生在同一時段做什麼不同的事。
  描述方式必須符合品質標準——畫面感、溫度、可操作。}

---

## 3. Check / Assessment（驗收評量）｜約 {n} 分鐘

{如何確認學生學到了什麼。可以是提問、快速寫作、小組分享等。
  說明不同層次學生的驗收標準差異。}

---

## 4. Closing（分享或回顧）｜約 {n} 分鐘

{如何結束這堂課。保持完整感。
  可以是全班分享、一句話回饋、預告下節課等。}

---

## 教師備忘
- 需要準備的材料：{清單}
- 預期的時間分配風險：{哪個環節可能超時，備案是什麼}
- 課後觀察紀錄提示：{觀察哪幾位學生、關注什麼}
```

# 2-B：差異化任務與驗收回饋機制設計

## 涵蓋範圍

| 面向 | 說明 |
|------|------|
| 課中差異化延伸 | 核心任務完成速度不同的學生，各自有預裝的延伸任務 |
| 家庭任務 | 培養英文學習習慣、可驗收、不加重負擔 |
| 減負驗收機制 | 透過同儕支援方式驗收，降低教師批改壓力 |

---

## 2-B-1：課中差異化延伸

### 設計邏輯

```
核心任務（所有學生都做）
  │
  ├─ 完成較快 → 延伸任務（預先設計好，不需老師即時指派）
  │     ├─ A 類：深化任務（換角度、更深的問題）
  │     └─ C 類：自選任務（多種輸出方式選一）
  │
  └─ 需要更多時間 → 繼續核心任務 + 輕量支持
        ├─ B 類：簡化版同任務（指引更明確、步驟更細）
        └─ D 類：配對互助 or 教師巡迴重點關注
```

### 設計原則

- 延伸任務必須**預裝**在教學流程中——寫在 Phase 3 產出的 Session 文件裡
- 延伸不是「多做一題」，而是「換一個角度再深入」
- 不讓快的學生覺得被罰做更多，也不讓慢的學生感覺被拋下
- 延伸任務對學生呈現為「做完的人可以挑戰這個」，不貼標籤

### AI 在 Phase 3 的產出要求

在每一節課的 `Main Activity` 段落中，必須包含：

1. **核心任務**：所有學生都做的基本活動
2. **延伸選項**：2-3 個不同方向的延伸任務（對應不同 DI 類型，但不明示標籤）
3. **支持策略**：B/D 類學生卡住時的具體介入方式

---

## 2-B-2：家庭任務設計

### 設計原則

| 原則 | 說明 |
|------|------|
| 習慣導向 | 目標是「每天接觸英文 10-15 分鐘」，不是完成一份報告 |
| 可驗收 | 有可見的完成痕跡，但不依賴老師逐份批改 |
| 差異化 | A/B/C/D 可以做不同版本，但在同一框架內 |
| 輕量 | 不加重學生與家庭的負擔 |

### 可能的任務型態

AI 在設計家庭任務時，可從以下型態中選擇或組合：

**微型任務（每日型）：**
- 每天一件小事，累積成習慣
- 例：抄三句喜歡的英文句子 / 聽一首英文歌寫下一個聽到的詞 / 讀一頁英文書

**累積式任務（每週型）：**
- 一週完成一個小作品
- 例：一段英文日記 / 一張手繪單字卡 / 一段 30 秒英文錄音

**自選菜單（選擇型）：**
- 給學生一張「本週任務菜單」，上面 5-6 個選項，選 2-3 個完成
- 不同選項自然涵蓋不同學習優勢與難度

### 產出格式

家庭任務隨每節課或每單元一起產出，附在 Session 文件的末尾，
或獨立為 `english-task-{n}-v{N}-{YYYYMMDD}.md`。

## 2-B-3：減負驗收機制

### 核心精神

老師只需要做「品管」（確認系統在運作），不需要逐件批改。

### 驗收機制選項

AI 在設計任務時，必須同時推薦一種驗收方式：

| 機制 | 運作方式 | 老師的角色 |
|------|---------|-----------|
| **配對檢核** | 兩人一組，互相確認任務完成並簽名。老師只看簽名表。 | 巡視，處理例外 |
| **小組展示** | 每週一次，4 人小組各用 2 分鐘分享作品。同組成員給一句回饋。 | 聽，偶爾追問 |
| **自我檢核表** | 學生自己勾選「完成 / 試了但卡住 / 沒做」。誠實不扣分，但必須填。 | 快速掃描，只追蹤「卡住」的人 |
| **A 帶 D 機制** | A 類學生的延伸任務之一是「協助一位同學完成任務」——這是更高層次學習。 | 確認配對，不介入過程 |

### 產出要求

每一份任務設計（課中延伸或家庭任務）都必須附帶：
1. 建議的驗收方式（從上表選擇或教師自訂）
2. 驗收所需的時間（通常不超過 5 分鐘）
3. 老師需要做的唯一動作（例如：收簽名表、掃一眼檢核表）
