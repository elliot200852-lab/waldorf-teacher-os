# 台灣主權 AI 訓練語料庫（TAIC）與 TeacherOS 協作分析

**日期：2026-03-25**
**分析者：AI 協作夥伴**
**目的：評估 TAIC 對 TeacherOS 的實質幫助，並提出具體協作方向**

---

## 一、TAIC 現況概覽

台灣主權 AI 訓練語料庫（TAIC，https://taic.moda.gov.tw）由數位發展部建置與營運，目前處於 Beta 階段。截至 2026 年 3 月，累計約 12 億 token、3,700 多筆資料集，涵蓋文化藝術、語言詞彙、歷史文物、在地文化、政府文件、學術內容等領域。資料來自中央研究院、國立美術館、原住民族委員會等 50 餘個機構。

授權模式分兩軌：開放資料（免註冊直接下載）與授權 AI 訓練資料（需註冊並同意授權條款）。資料格式以 JSON 為主，附帶多語後設資料。TAIC 同時也是 TAIDE（可信任生成式 AI 發展計畫）的語料基礎，最新模型為 2026 年 2 月釋出的 Gemma-3-TAIDE-12b-Chat。

---

## 二、TeacherOS 現有資料架構的缺口

TeacherOS 目前不內建靜態語料庫，而是依賴即時 API 研究（Met Museum、Europeana、YouTube、Wikipedia）。這個設計的優勢是資料即時且經過驗證，但存在三個結構性缺口：

1. **台灣本土素材的系統性不足。** story-research 技能從 Wikipedia 與歷史記錄取材，但缺少結構化的台灣文化、歷史、語言資料庫作為底層支撐。素材的深度與覆蓋率取決於每次 API 查詢的運氣。

2. **正體中文語境的貧乏。** 英文教學中需要的跨文化對照素材（台灣視角看世界、世界視角看台灣），目前沒有可靠的正體中文知識庫可供引用。

3. **原住民族與在地文化的空白。** 歷史課從台灣主體意識出發，需要原住民南島文化、多重殖民歷史的一手素材，這些在現有 API 來源中極度稀缺。

---

## 三、TAIC 對 TeacherOS 的具體幫助

### 3.1 歷史課：台灣主體意識的素材基礎

TAIC 收錄的歷史文物、在地文化資料集，直接對應 TeacherOS 的歷史教學哲學核心——從台灣主體意識出發，以全球史觀取代歐洲中心史觀。

具體應用場景：

- **七年級地理大發現單元：** TAIC 的歷史文物類資料可能包含荷蘭時期、漢人移民相關的一手文獻或文物描述，讓「Formosa 三視角敘事」從教師口述升級為有文獻依據的現象學教學。
- **八至九年級革命與現代化：** 日治時期現代化、二二八、白色恐怖的原始文獻或口述歷史，若在 TAIC 中有收錄，可直接成為傳記式教學的素材來源。
- **原住民族素材：** 原住民族委員會是 TAIC 合作機構之一，其貢獻的南島文化、族語、口傳文學資料，填補的正是 TeacherOS 歷史課程中最難取得的素材缺口。

### 3.2 英文課：正體中文→英文的跨文化橋樑

華德福英文教學的目標不只是語言技能，而是透過語言認識另一種世界觀。TAIC 的正體中文語料可以作為「出發點」——學生先在母語中建立對台灣文化的深刻理解，再透過英文表達與翻譯練習，將這些理解轉化為跨文化溝通能力。

具體應用場景：

- **文化藝術類語料 → 英文描述寫作：** 從 TAIC 取得台灣藝術作品的正體中文描述，設計「用英文向外國人介紹這件作品」的寫作任務，同時鍛鍊語言能力與文化認同。
- **語言詞彙類語料 → 語言比較教學：** TAIC 的詞彙與術語資料集可用於設計中英語言結構比較的教學活動，讓學生體驗施泰納所說的「每一種語言以獨特方式塑造說話者的內在生命」。
- **poetry-research 技能的延伸：** 將 TAIC 的文學類語料納入詩歌研究流程，讓英文詩歌教學能夠與台灣文學作品進行對照閱讀。

### 3.3 差異化教學：本土化素材的分層運用

TeacherOS 的差異化教學以能力×動機雙軸矩陣為核心。TAIC 的多元資料集天然適合分層設計：

- **A 型學生（高能力高動機）：** 直接閱讀 TAIC 原始文獻（學術類、歷史類），進行獨立研究與批判分析。
- **B 型學生（高能力低動機）：** 以 TAIC 中趣味性較高的在地文化素材作為引入點，重新點燃學習興趣。
- **C 型學生（低能力高動機）：** 將 TAIC 素材簡化改寫，保留核心內容但降低語言難度。
- **D 型學生（低能力低動機）：** 以 TAIC 的視覺類素材（藝術圖像、文物照片）為錨點，減少文字負擔，先建立情感連結。

### 3.4 story 系列技能的語料升級

TeacherOS 的「臺灣的故事」系列技能（story-planner → story-research → story-writer → story-verify → story-archive）目前依賴 API 搜尋與 AI 合成。TAIC 可以作為 story-research 的優先查詢來源，提供經過政府與學術機構驗證的台灣故事素材，大幅提升事實正確性（story-verify 的通過率）。

## 四、協作建議：具體行動方案

### 方案 A：TAIC 作為 TeacherOS 的參考資料來源（短期，可立即執行）

**做法：** 在 `ai-core/reference/` 新增一個 `corpus-taic.yaml` 知識模組，記錄 TAIC 的存取方式、資料分類、授權條件，讓 AI 在備課時知道可以查詢 TAIC。在相關技能（story-research、poetry-research、subject-lesson-45）中加入 TAIC 查詢步驟。

**成本：** 低。只需更新 YAML 與技能文件，不需要下載或儲存任何語料。

**效果：** AI 備課時多一個可靠的台灣本土素材來源，特別是歷史與文化領域。

### 方案 B：建立 TeacherOS 教學語料子集（中期，需評估授權）

**做法：** 從 TAIC 下載與七至九年級教學直接相關的資料集（歷史文物、在地文化、原住民族、文學藝術），整理為 TeacherOS 可用的結構化格式，存放在 `ai-core/corpus/taic-education/` 目錄下。

**關鍵考量：**
- 需確認 TAIC 授權條款是否允許「教育用途的二次整理與儲存」
- 開放資料軌的內容可自由再製與改作
- 授權 AI 訓練資料軌需遵守特定條款

**成本：** 中。需要人工篩選、格式轉換、定期同步更新。

**效果：** 離線可用的台灣教學素材庫，不依賴即時網路查詢。

### 方案 C：向 TAIC 貢獻教育領域語料（長期，具社會影響力）

**做法：** TeacherOS 在運作過程中產生大量結構化的教學設計、課程敘事、學生發展描述。這些以華德福教育哲學為基礎、用正體中文撰寫的教育內容，本身就是高品質的正體中文語料。可以考慮將去識別化的課程設計範本與教學敘事，以開放資料形式回饋給 TAIC。

**意義：** 目前 TAIC 的 12 億 token 中，教育領域（特別是非體制教育、實驗教育、華德福教育）的語料幾乎為零。TeacherOS 的貢獻將填補這個空白，讓未來的 AI 模型更理解台灣多元教育生態。這完全符合 David 作為「意義架構師」的定位——不只使用 AI，更定義 AI 應知道什麼。

**成本：** 高。需要設計去識別化流程、品質審核機制、授權框架。

**效果：** 台灣 AI 模型更理解華德福教育與實驗教育脈絡；TeacherOS 從 TAIC 的使用者升級為貢獻者，建立與數發部的實質合作關係。

## 五、風險與注意事項

1. **語料品質的異質性。** TAIC 目前仍在 Beta 階段，3,700 筆資料集的品質參差不齊。用於教學前必須經過人工審核，不能盲目信任。

2. **華德福教育的特殊需求。** TAIC 的設計目的是通用 AI 訓練，不是教育用途。其中的歷史敘事可能採用體制內史觀（或未經充分考量台灣主體性），需要教師以批判眼光篩選。這反而是一個教學機會——讓學生比較不同來源的歷史敘事，培養判斷力。

3. **授權條款的灰色地帶。** 「教育用途」與「AI 訓練用途」在 TAIC 的授權框架中可能有不同解釋。在方案 B 之前，建議先聯繫數發部確認。

4. **資料量的期望管理。** 12 億 token 聽起來很多，但分散在 3,700 筆資料集中，每筆平均只有約 32 萬 token。與七至九年級教學直接相關的可能只是其中一小部分。

---

## 六、結論

TAIC 對 TeacherOS 最大的價值不在於「大量資料」，而在於**它是目前唯一由政府背書、結構化收錄台灣本土文化與歷史素材的正體中文語料庫**。這正好填補了 TeacherOS 最核心的素材需求——台灣主體意識的歷史教學、跨文化英文教學、原住民族與在地文化。

建議的優先順序：方案 A（立即執行）→ 方案 B（本學期評估）→ 方案 C（長期規劃）。

方案 C 的意義超越 TeacherOS 本身：當台灣的主權語料庫中開始出現華德福教育的語料，就意味著 AI 世界裡多了一種「真正人類特質」的聲音。這不就是意義架構師該做的事嗎——決定 AI 應知道什麼。
