# TeacherOS 新手上路指南——行政角色

> 本文件適用於行政教師（無帶班、無授課，負責學校行政業務）。
> 新教師加入時，由管理員根據角色寄送對應版本。

---

歡迎加入 TeacherOS。你的情況比較特別——沒有課、沒有帶班，所以系統裡現成的「備課」模板對你來說不太適用。

但這恰恰是你可以大展身手的地方。

## TeacherOS 不只是備課系統

這套系統的底層其實是一個「工作追蹤 + AI 協作」的架構。核心機制很簡單：

1. 把一件事情框起來，給它一個名字（專案資料夾）
2. 記錄做到哪、下一步是什麼（進度錨點）
3. 每次開工，AI 自動接續上次進度，不用重新解釋

這套機制用在備課，就是課程設計系統。用在行政工作，就是行政專案管理系統。結構一樣，裝什麼內容由你決定。

## 你的行政工作可以怎麼用

以下是一些可能的方向，挑跟你工作相關的：

- 學校活動企劃：擬活動計畫書、時程表、分工表、場地安排、物資清單
- 會議紀錄與追蹤：整理會議重點、追蹤待辦事項、提醒未完成的決議
- 公文與文件處理：起草公文、通知、信件，整理歸檔
- 招生或對外溝通：擬招生說明文案、家長說明會流程、校刊稿件
- 學校行事曆：整理年度行事曆、活動排程、衝突檢查
- 資料統計與報告：整理學生資料、出缺勤統計、向上呈報的報告
- 教師研習規劃：排定研習時程、整理講師資訊、彙整回饋
- 經費或採購管理：製作預算表、核銷清單、報價比較

## 怎麼開始

直接跟 AI 說就好。例如：

「我是行政老師，沒有帶班。我想建一個專案來管理學校的活動企劃。」

AI 會在你的空間裡建立一個專案資料夾，裡面有進度追蹤檔。之後每次開工，AI 記得你做到哪裡，自動接續。

你可以一次建多個專案，例如活動企劃、公文文件、教師研習、招生，每個專案獨立追蹤進度。

## 成立專案後，AI 會幫你做什麼

1. 在你的 workspace 裡建立專案資料夾（例如 projects/school-events/）
2. 產生進度追蹤檔（session.yaml），記錄你做到哪裡、下一步是什麼
3. 每次開工自動讀取進度，報告現況
4. 對話結束時自動更新進度，下次無縫接續

你的行政專案結構可以像這樣：

```
projects/
  school-events/       <- 活動企劃
  admin-docs/          <- 公文與文件
  teacher-training/    <- 教師研習
  recruitment/         <- 招生相關
```

每個專案都是獨立的工作線，各有各的進度錨點。

## 關於設定

你的 workspace 裡有一份 teacheros-personal.yaml。因為你不是科任老師，裡面的「科目信念」可以跳過不填。重點填：

- 你的角色（行政教師 / 你的正式職稱）
- 你的工作偏好（例如：處理大量文書、需要表格整理、喜歡條列式溝通）

或者直接對 AI 說：「幫我設定 teacheros-personal.yaml，我是行政老師，主要負責___、___、___。」AI 會幫你填，不用自己改檔案格式。

## 最重要的一件事

模板裡沒有的東西，不代表系統不支援——那正是你可以自己創造的空間。

AI 不只會幫人備課，它會幫你優化任何有結構的工作。你定義工作內容，AI 幫你追蹤、整理、產出、記憶。你的行政工作越有結構，AI 就越能幫上忙。

## 日常口令

- 說「開工」：AI 載入系統，報告你的進度
- 說「收工」：儲存進度，下次接續

不需要記精確指令，用你的話說就好。

## 遇到不會的時候怎麼辦

直接問 AI。例如：

- 「我想整理一份活動企劃，要從哪開始？」
- 「幫我追蹤上次會議的待辦事項」
- 「這份公文的格式對不對？」
- 「我想做的事情，系統有支援嗎？」

AI 已經載入了完整的系統文件，把它當成一個熟悉系統的同事來問就好。

## 建議節奏

- 第一天：跟 AI 說你是誰、負責什麼，讓它幫你建第一個專案
- 第一週：挑一件手邊的行政工作，試著請 AI 幫你整理
- 之後：慢慢擴展，把更多行政工作線納入系統
