# TeacherOS 英文課備課系統｜教師試用手冊

> 版本：試用版（Beta）
> 適用科目：英文
> 適用年級：七、八、九年級
> 聯絡人：David（系統設計者）

---

## 一、這套系統能幫你做什麼

TeacherOS 是一套讓 AI 協助備課的工作系統。你不需要懂任何程式，只需要用說話或打字的方式，跟 AI 一起完成以下工作：

- **學季教學大綱**：一學年的整體規劃，含教學目標、策略、評量設計
- **單元教學流程**：每一個教學單元的 45 分鐘課堂流程
- **差異化任務設計**：針對不同學習樣態的學生，設計分層任務

完成後，系統可以把文件輸出成 Google Docs，供你分享或列印。

## 二、你需要準備什麼

### 帳號與工具（第一次設定，之後不用再做）

| 項目 | 說明 | 取得方式 |
|------|------|----------|
| GitHub 帳號 | 用來下載系統資料夾 | 前往 [https://github.com](https://github.com) 免費申請 |
| AI 工具（擇一） | 進行備課對話的 AI 工具 | 見下方說明 |
| Google 帳號 | 備課文件會備份到這裡 | 你已有的個人或學校帳號均可 |
| Google Drive for Desktop | 讓 Google Drive 在你的電腦上同步 | 見下方安裝說明 |
| Pandoc | 文件格式轉換工具（在後台執行，你不會看到它） | 見下方安裝說明 |

---

### 選擇你的 AI 工具

本系統支援任何 AI 工具進行備課對話。請依照你的習慣或學校提供的授權，選擇以下其中一種：

**(a) ChatGPT**
前往 [https://chatgpt.com](https://chatgpt.com)，登入或申請帳號即可使用。

**(b) Gemini（搭配 Antigravity）**
前往 [https://gemini.google.com](https://gemini.google.com)，使用你的 Google 帳號登入。
若學校提供 Antigravity 工具，可搭配使用以取得更完整的功能。

**(c) Claude Code（建議，功能最完整）**
Claude Code 可直接在本機執行指令，支援文件自動輸出到 Google Drive。
請向 David 取得帳號或邀請連結。

> **注意：** 三種工具在備課對話方面功能相近。若使用 (a) 或 (b)，文件輸出到 Google Drive 的步驟需要手動完成；使用 (c) 則可由 AI 自動執行。

---

### 取得系統資料夾

1. 確認你已有 GitHub 帳號（尚未申請請先至 [https://github.com](https://github.com) 申請）
2. 向 David 確認你已取得 Repo 的存取權限
3. 打開終端機，執行以下指令，將系統資料夾下載到你的電腦：

```bash
git clone https://github.com/elliot200852-lab/waldorf-teacher-os.git
```

4. 完成後，系統資料夾會出現在你目前的位置

> **網路問題：** 如果無法連線至 GitHub，或遇到防火牆、VPN 等網路問題，請聯絡學校的網路工程師協助處理。

---

### 安裝 Google Drive for Desktop

1. 前往 [https://www.google.com/drive/download/](https://www.google.com/drive/download/)
2. 下載並安裝
3. 登入你的 Google 帳號
4. 完成後，你的電腦上會出現一個 Google Drive 資料夾

---

### 安裝 Pandoc

打開「終端機」（Mac：應用程式 → 工具程式 → 終端機），貼上以下指令，按 Enter：

```bash
brew install pandoc
```

如果出現「找不到 brew」，請先執行以下指令，安裝完成後再重新執行上方指令：

```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```

---

### 建立你的個人設定檔

在系統資料夾內，找到 `setup/environment.env.example`，複製一份，改名為 `environment.env`，用文字編輯器打開，填入你的個人資訊：

```
USER_NAME=你的姓名
USER_EMAIL=你的Email
GOOGLE_DRIVE_EMAIL=你登入 Google Drive 的帳號
GOOGLE_DRIVE_FOLDER=00-01-TeacherOS-專案三層記憶
GITHUB_USERNAME=你的GitHub帳號名稱
PANDOC_PATH=/opt/homebrew/bin/pandoc
```

---

### 確認環境設定完成

打開終端機，進入系統資料夾，執行：

```bash
bash setup/setup-check.sh
```

全部顯示 `[OK]` 就可以開始了。有 `[!!]` 的項目，請對照提示處理。

## 三、改寫屬於你自己的 TeacherOS

**這是最重要的一步。**

目前系統內的 `ai-core/teacheros.yaml` 是 David 的版本，記錄了他的教學哲學、工作偏好與班級資訊。在你開始備課之前，你必須依照自己的情況改寫這份文件，讓 AI 認識「你是誰、你教什麼、你的學生是誰」。

### 需要改寫的內容

打開 `ai-core/teacheros.yaml`，依照以下方向修改：

- **教師基本資訊**：你的姓名、任教年級、科目
- **教學哲學**：你對英文教育的核心理念（用自己的語言描述）
- **工作偏好**：你習慣的備課方式、偏好的 AI 協作風格
- **班級資訊**：你的班級代碼、人數、大致的學習樣態

### 改寫後建立你的班級資料夾

在 `projects/` 資料夾內，依照你的班級代碼建立對應的資料夾結構：

```
projects/
  class-{你的班級代碼}/
    working/
    content/english/
    english/content/
```

你可以複製現有的 `class-9c/` 資料夾結構，再修改內容。

> 如果對改寫有疑問，可以先把你的班級背景和教學理念說給 AI 聽，請 AI 協助你草擬 teacheros.yaml 的內容。

## 四、每次使用的流程

### 開始一次對話

打開你選擇的 AI 工具，進入系統資料夾，然後直接對 AI 說：

> 「我要開始做 {你的班級代碼} 英文課學季大綱。」

或者使用快速指令：

> `/syllabus`

AI 會自動載入系統脈絡，問你幾個關於班級和課程的問題，然後帶你一步步完成。

**你不需要解釋這套系統是什麼，也不需要重新介紹自己。AI 已經知道。**

### 中途離開，下次繼續

每次對話結束，AI 會記錄你做到哪裡。下次直接說：

> 「我要繼續上次 {班級代碼} 英文課的工作。」

AI 會告訴你上次停在哪裡，然後繼續。

### 如果感覺對話偏離了

直接說：

> 「回到中軸。」

AI 會立刻重新定位，告訴你現在在哪個步驟，問你要繼續還是調整。

## 五、備課工作的兩個階段

### 階段一：學季整體規劃（Block 1）

**指令：`/syllabus`**

產出：一份完整的學季教學大綱，包含：
- 教學目標（條列式，適合給家長閱讀）
- 教學策略（說明你的教學方式與理念）
- 學期規劃（各單元主題與時程）
- 評量設計（含評分比例）

這份大綱是整個學期備課的基準。建議先完成這個，再進行階段二。

### 階段二：課堂教學設計（Block 2）

**指令：`/lesson`**

產出：
- 2-A：單元教學流程（45 分鐘四段結構：暖身→主活動→檢核→收尾）
- 2-B：差異化任務設計（針對不同學習樣態學生的分層任務）

## 六、把文件輸出到 Google Docs

每次 AI 完成一份文件並確認版本後，AI 會問你：

> 「這份文件要輸出到 Google Drive 嗎？」

你只需要回答**「要」**。

**使用 Claude Code 的老師：** AI 會自動完成轉換，文件直接出現在你的 Google Drive。

**使用 ChatGPT 或 Gemini 的老師：** 請把 AI 產出的文字內容，複製貼上到 Google 文件，手動儲存。

文件完成後，在 Google Drive 找到對應檔案，右鍵 → 「用 Google 文件開啟」，即可分享或列印。

---

## 七、關於差異化教學

這套系統使用「能力 × 動機」雙軸矩陣，將學生分為四類（A/B/C/D），針對各類學生設計不同的學習任務和支持方式。

**分類邏輯：**
- A 類：高能力、高動機
- B 類：高能力、低動機
- C 類：低能力、高動機
- D 類：低能力、低動機

**重要原則：分類標籤只有老師和 AI 看得到，學生看不到。**

AI 在設計差異化任務時，會先描述各類學生的內在狀態，再提出具體的課堂做法，讓你讀完後腦海能浮現實際的課堂場景，可以直接拿來用。
