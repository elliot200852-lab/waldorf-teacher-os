# YAML 管結構、Markdown 管內容：一個老師的身份被翻譯成機器語言之後

接著昨天的話題。我提到用一種叫 YAML 的格式寫了一份「教師身份定義檔」，今天來揭曉它長什麼樣子。不過在那之前，先說一件讓我自己也覺得荒謬的事。

我花了超過十年建立的教學信念——關於青少年的發展階段、關於課程設計的哲學、關於差異化教學為什麼不是選項而是義務——這些東西在我腦袋裡住了很久。但當我試著把它們寫成 YAML 的時候，才發現：有些我以為自己很清楚的事情，其實一直是模糊的。

比如，我一直覺得自己「重視差異化教學」。但當 YAML 要求我精確定義「差異化的維度是什麼」的時候，我楞住了。是依照能力分？動機分？學習風格分？還是全部都要？如果全部都要，優先順序是什麼？

YAML 不接受含糊。它就是一個極度挑剔的清單格式。你寫「能力」，它就記住「能力」。你沒寫「動機」，它就不知道動機的存在。（有點像某些很literal的學生，你說什麼他就信什麼，你沒說的他真的不會自己猜。）

這種強迫性的精確，一開始讓我不舒服。教育本來就充滿灰色地帶，硬要塞進結構化格式，感覺像在冒犯什麼。但做著做著我開始理解——YAML 不是在消滅灰色地帶，它是在幫你畫出灰色地帶的邊界。你可以在框架裡保留彈性空間，但那個彈性必須是你刻意設計的，不是你因為沒想清楚所以放任的。

讓我用一個簡化的例子，給你看它大概長什麼樣子：

```
teacher_identity:
  name: David
  role: 華德福教師
  school_type: 體制外實驗學校

teaching_scope:
  grades: [七年級, 八年級, 九年級]
  subjects: [英文, 歷史, 跨領域主題課程]

subject_philosophy:
  english:
    belief: 語言不只是溝通工具，是塑造思維的力量
    approach: 以文學文本觸碰靈魂，不是技能訓練

differentiated_instruction:
  mandate: 每堂課的必要條件，不是選項
  axes: [能力, 動機]
```

就是這樣。沒有花俏的程式碼，沒有演算法。只是把「我是誰、我怎麼教」用一種有層次的清單寫出來。冒號左邊是問題，右邊是答案。縮排代表從屬關係。一個完全不會coding的老師看得懂嗎？我覺得可以——至少比看 Python 友善太多了（到現在我也只知道它怎麼唸）。

這是 TeacherOS x CreatorHub 的第一層——身份層。AI 每次開工前先讀這個檔案，就像一個新同事到職第一天先讀你的教學手冊。但光有身份還不夠，你還需要知道「現在這個專案在哪」——這學期上到哪了？這個教學區塊完成了嗎？學生的狀態怎麼樣？這是第二層，專案層。然後是第三層：今天要推進哪一條工作線？是備課、是寫家長信、還是處理上週的班級衝突？這是工作層。

三層疊起來，就像一棟房子。身份層是地基——不常動，但所有東西都蓋在上面。專案層是樓層結構——每個學期調整。工作層是每天的家具擺設——天天在動。

YAML 管的是這些層的結構。但教學的實際內容——備課筆記、教學大綱、學生觀察紀錄、課堂反思——這些用 Markdown 寫。Markdown 就是一種簡單的文字格式，你用過任何筆記軟體大概都碰過（其實你現在看到的這篇文章，原始檔就是 Markdown）。

YAML 管骨架，Markdown 管血肉。一個負責讓 AI 知道「該看哪裡」，一個負責讓 AI 知道「看到之後該理解什麼」。聽起來很技術，但它解決的問題非常人文：一個老師的教學智慧，怎麼從「只存在我腦袋裡」變成「任何 AI 都能讀取並延續」？

這不是抽象的問題。有一位老師，教書三十年。他知道怎麼在一個孩子快要爆炸的時候，用一個眼神讓整間教室安靜下來。他知道八年級的第三週永遠會出事，因為新鮮感剛過、秩序感還沒建立。這些東西不在任何教科書裡——它們活在他的身體裡、他的節奏裡、他走進教室的方式裡。但他退休那天，全部跟著他走了。下一個接手的人拿到一疊教案、一個電腦資料夾，然後從零開始。

TeacherOS x CreatorHub 想解決的，不只是「讓 AI 記住我」，而是「讓教學智慧可以被結構化地傳承」。聽起來野心很大，但起點其實很小——就是一個老師坐下來，誠實地把自己寫清楚。寫清楚自己這件事，比你想像的困難。但也比你想像的值得。你可能會在過程中第一次發現，原來有些你以為是信念的東西，其實只是不自知的習慣，或是一種直覺，或是一種無奈。

或許「老師」這個行當，有點像神話中的薛西弗斯。我們薛西弗斯們要樂觀地相信「薛西弗斯是快樂的。」所以有想搞點什麼的衝動要把握，我們可以每天快樂地推石頭！

我是 David，在臺灣一所華德福學校教書。我正在做一件必須重複但有點意思的事——找到跟AI對話的語言重新定義一個老師的工作方式。Chapters to be continued...
