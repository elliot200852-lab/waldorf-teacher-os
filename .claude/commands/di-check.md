# /di-check — DI 雙軸合規檢查

對當前對話中的課程設計進行差異化教學合規核對。
只標記未滿足的項目，不貼出完整規則。

## 使用方式

```
/di-check
```

不需要班級參數。直接掃描當前對話中最近一份課程設計內容。

## 執行步驟

### Step 1 — 讀取 DI 規則

讀取：`/Users/Dave/Desktop/WaldorfTeacherOS-Repo/projects/_di-framework/project.yaml`

提取以下兩個強制軸線的要求：
- `template_requirements.mandatory_axes`
- `di_axes.axis_learning_strengths.dimensions`
- `di_axes.axis_ability_motivation.matrix`

### Step 2 — 掃描當前對話

從本次對話中找出最近一份課程設計或評量設計內容，進行以下核對：

**軸線一：學習優勢覆蓋（至少 2–3 種）**
- [ ] 語言型入口（閱讀、文字、口語討論）
- [ ] 圖像型入口（圖表、視覺化、素描記錄）
- [ ] 動作型入口（身體移動、實作、手工）
- [ ] 藝術型入口（音樂、繪畫、戲劇、詩歌）

**軸線二：A/B/C/D 矩陣覆蓋（至少高挑戰 + 鷹架兩層）**
- [ ] A 類（高能力高動機）有延伸挑戰與自主空間
- [ ] B 類（低能力高動機）有鷹架支持與成功經驗
- [ ] C 類（高能力低動機）有意義連結與興趣切入
- [ ] D 類（低能力低動機）有安全感設計與具體操作入口

### Step 3 — 輸出結果

只輸出**未滿足**的項目。格式如下：

---

**DI 核對結果**

軸線一｜學習優勢：[通過 / X 種，缺少：列出缺少的類型]
軸線二｜A/B/C/D：[通過 / 缺少：列出未涵蓋的類型]

**需要補強的方向：**
（針對每個缺口，給出 1 句具體建議，不超過 3 個建議）

---

若兩個軸線都通過，輸出：「DI 雙軸核對通過。」

## 注意事項

- 不重複貼出已通過的項目
- 建議簡短具體，不展開說明
- 核對範圍是課程「設計層次」，不延伸評估教師實際執行能力
