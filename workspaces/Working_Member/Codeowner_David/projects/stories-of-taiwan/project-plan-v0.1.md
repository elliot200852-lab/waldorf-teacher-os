# 「臺灣的故事」專案計畫書 v0.1

> **定位**：獨立第三條工作線，長期累積型教學素材生產系統
> **發起人**：David
> **日期**：2026-03-21
> **狀態**：草稿，待 David 審閱

---

## 一、專案願景

每天產出一篇「臺灣的故事」——精煉的文字、動人的畫面、有憑有據的史料——長期堆疊成一座教育的搖籃。學生在這個環境中持續沉浸，自然培養對臺灣土地的主體意識。

不是一次性的教案，而是一座慢慢長出來的故事庫。

---

## 二、核心設計原則

1. **有憑有據**：每篇故事必須錨定在至少一個可溯源的真實資料（檔案、照片、地圖、文物、口述）
2. **主題先行**：先有大區塊骨架，篇章在脈絡中生長，不是孤立的每日隨機
3. **課程對齊**：以一年為單位、四季為節奏，配合五年級臺灣史地主課程的教學進度
4. **三件套產出**：每篇故事固定產出文字內容 + 教師說書稿 + 投影圖像參考
5. **跨齡可擴展**：從五年級起步，未來可向上延伸至六到九年級（深度遞增）

---

## 三、主題區塊骨架（一年四季）

以五年級「臺灣史地」主課程為主軸，配合華德福第二個七年週期（7-14 歲）的學習特質——圖像、故事、情感連結為認知媒介。

### 秋季學期（9-11 月）——「這片土地的記憶」

**區塊 A：島嶼的誕生與最初的人（史前—原住民）**
- 臺灣的地質形成、山脈與海岸
- 史前遺址（卑南、十三行、大坌坑）
- 南島語族的遷徙與定居
- 各族群的土地故事（泰雅的獵場、阿美的海、排灣的百步蛇）
- 預估篇數：15-20 篇

**區塊 B：海上來的人（大航海時代 1540s-1620s）**
- 葡萄牙人眼中的 Formosa
- 東亞海域的貿易網絡
- 臺灣在世界地圖上的出現
- 早期漢人與日本商人的足跡
- 預估篇數：8-12 篇

### 冬季學期（12-2 月）——「被書寫的島嶼」

**區塊 C：紅毛城的時代（荷蘭與西班牙 1624-1662）**
- 荷蘭東印度公司在大員（臺南）
- 西班牙人在雞籠（基隆）與淡水
- 荷蘭人與原住民的互動（新港文書）
- 鄭芝龍與海上勢力
- 預估篇數：12-15 篇

**區塊 D：國姓爺與他的後人（鄭氏時期 1662-1683）**
- 鄭成功的傳奇——英雄敘事適合十歲兒童的崇拜階段
- 從軍事佔領到農墾開發
- 鄭經與陳永華——制度建設
- 明鄭的終結與臺灣的轉折
- 預估篇數：10-12 篇

### 春季學期（3-5 月）——「帝國邊陲的日常」

**區塊 E：清帝國下的臺灣（1683-1895）**
- 渡海禁令與偷渡客
- 械鬥、開墾與水利（瑠公圳、曹公圳）
- 茶、樟腦、糖——臺灣的物產與世界
- 原漢關係與「番界」
- 林爽文事件、戴潮春事件——反叛與正義
- 劉銘傳的現代化——鐵路、電報、郵政
- 預估篇數：20-25 篇（最長區塊，內容最豐富）

### 夏季學期（6-7 月）——「新的開始」

**區塊 F：日本時代的臺灣（1895-1945）**
- 馬關條約與乙未戰爭
- 基礎建設——鐵路、公路、水庫、學校
- 霧社事件——莫那魯道
- 嘉南大圳與八田與一
- 日常生活——市場、學校、神社
- 預估篇數：15-20 篇

**區塊 G：我們的時代（1945— ）**
- 光復與二二八
- 白色恐怖與戒嚴
- 經濟奇蹟
- 民主化運動
- 預估篇數：10-15 篇（五年級點到為止，留給高年級深化）

**全年預估：90-120 篇**（以學期週數計算，扣除假期約 180 個上課日，每日一篇綽綽有餘，可保留彈性日做回顧與串接）

## 四、單篇故事產出模板

每篇故事固定產出三份文件：

### 4.1 文字內容（story-content.md）

```
# [編號] [故事標題]
> 區塊：[A-G] | 主題：[子主題] | 年代：[西元/朝代]

## 故事本文
（300-500 字，敘事體，有場景、有人物、有感官細節）

## 史實錨點
- 資料來源 1：[名稱]（[資料庫代碼]）[連結]
- 資料來源 2：...
- 圖像來源：[出處與授權]

## 地理標記
- 地點：[地名]
- 經緯度：[座標]（如有）
- 現今對應：[現在這個地方叫什麼/長什麼樣]

## 延伸線索
- 與前一篇的連結：...
- 可延伸方向：...
- 跨齡提示：（六年級可加深什麼、九年級可延伸什麼）
```

### 4.2 教師說書稿（story-narration.md）

```
# [編號] 教師說書稿：[故事標題]

## 開場（1-2 分鐘）
（口語化的引入，可包含提問或回顧前一天）

## 說書本文（5-8 分鐘）
（逐字稿，語氣標註，停頓標記，聲音變化提示）
（適合朗讀節奏，含華德福說故事的「吸氣-呼氣」節奏）

## 圖像展示時機
- [時間點 1]：展示 [圖像 A 描述]
- [時間點 2]：展示 [圖像 B 描述]

## 收尾（1 分鐘）
（留下懸念或連結明天的線索）

## 教師備註
- 可能的學生提問與回應方向
- 情感注意點（哪些內容可能觸動學生）
- 差異化提示（A/B/C/D 學生的不同回應方式）
```

### 4.3 投影圖像清單（story-images.md）

```
# [編號] 圖像參考：[故事標題]

## 主圖
- 檔案：[路徑或 URL]
- 來源：[資料庫名稱]
- 授權：[CC / 教育用途 / 需另行確認]
- 說明：[這張圖在故事中的角色]

## 輔助圖（地圖/文物/建築）
- ...

## 投影建議
- 建議投影順序與時機（對應說書稿的時間點）
- 若無法投影的替代方案（實體列印 A3）
```

## 五、資料源盤點與可用性評估

### 第一層：有程式化 API，可自動搜尋

| 資料源 | 代碼 | API 狀態 | 需要 Key | 內容特色 |
|--------|------|----------|----------|----------|
| 國家文化記憶庫 TCMB | tcmb | 有 Swagger，待 Key 下來 | 是（已申請） | CC 授權文化素材，圖文豐富 |
| 政府資料開放平臺 | datagov | 有 REST API | 否 | 結構化資料集（古蹟、228 紀念碑等） |
| 文化資產國家網路 BOCH | boch | 端點回 404，疑似遷移 | 否 | 古蹟/歷史建築/考古遺址 |

**BOCH 狀態待確認**：`data.boch.gov.tw/opendata/v2/` 目前回 404，可能已遷移至 data.gov.tw 或新網域。需手動查證最新 API 文件。

### 第二層：有網站可瀏覽，需 AI 輔助研究

| 資料源 | 代碼 | 網站狀態 | 內容特色 |
|--------|------|----------|----------|
| 國家檔案資訊網 AA | archives | 運作中 | 228/白色恐怖政治檔案，數位影像可線上瀏覽 |
| 臺灣記憶 TM | tm | 運作中 | 老照片、碑碣、古書契、日治明信片、地方志 |
| 開放博物館 OpenMuseum | openmuseum | 運作中 | 中研院跨館策展，社群貢獻內容 |

### 第三層：需確認現況

| 資料源 | 代碼 | 狀態 | 備註 |
|--------|------|------|------|
| 國家人權博物館 NHRM | nhrm | 主站有重導向 | 記憶庫子站（hrmtw.nhrm.gov.tw）可能離線或遷移 |

### 補充資料源（建議新增）

- **中研院臺灣史研究所**：臺灣日記知識庫、臺灣總督府檔案
- **國立臺灣歷史博物館**：除 TCMB 外的自有典藏系統
- **各縣市文化局文資資料**：在地故事的第一手來源
- **維基共享資源**：大量 CC 授權臺灣歷史圖像

## 六、每日生產管線設計

### 6.1 管線流程

```
[每日觸發] Scheduled Task
    │
    ├── Step 1: 選題（story-planner skill）
    │   ├── 讀取年度主題骨架 → 確認今天在哪個區塊
    │   ├── 讀取已完成故事索引 → 避免重複
    │   └── 產出：今日故事主題 + 搜尋關鍵字
    │
    ├── Step 2: 資料搜尋（story-research skill）
    │   ├── 呼叫 taiwan_history_api.py 搜尋多資料源
    │   ├── AI 補充研究（網路搜尋、交叉比對）
    │   └── 產出：原始素材包（文字 + 圖像連結 + 來源標註）
    │
    ├── Step 3: 故事撰寫（story-writer skill）
    │   ├── 依模板產出三件套（內容 + 說書稿 + 圖像清單）
    │   ├── 事實查核：每個史實錨點必須有來源
    │   └── 產出：三份 .md 文件
    │
    ├── Step 4: 品質檢查（story-verify skill）
    │   ├── 確認史實錨點有效性
    │   ├── 確認與前後篇的敘事連貫
    │   ├── 確認差異化提示完整
    │   └── 產出：通過 → 歸檔；未通過 → 標記待修
    │
    └── Step 5: 歸檔與索引更新（story-archive skill）
        ├── 存入對應區塊資料夾
        ├── 更新故事索引（含標籤、地理、時間軸）
        └── 產出：更新後的 index.yaml
```

## 七、所需 Skills 規劃

### 7.1 新建 Skills（4 個核心 + 1 個排程）

| Skill 名稱 | 觸發語 | 功能 |
|------------|--------|------|
| `story-planner` | 「今天的故事」「選題」「下一篇」 | 根據骨架選定今日主題，產出搜尋關鍵字 |
| `story-research` | 「搜尋素材」「找資料」「查史料」 | 呼叫 API + AI 研究，產出原始素材包 |
| `story-writer` | 「寫故事」「產出故事」「生成三件套」 | 將素材精煉為三份產出文件 |
| `story-verify` | 「檢查故事」「品質確認」 | 事實查核 + 敘事連貫 + 格式合規 |
| `story-archive` | 「歸檔」「更新索引」 | 存檔 + 索引更新 |

### 7.2 Scheduled Task

一個每日排程任務，串接上述五個 Skill，自動執行 Step 1-5。
David 可選擇：
- **全自動模式**：每日自動跑完整管線，David 隔天審閱
- **半自動模式**：排程只跑到 Step 2（選題 + 搜素材），David 確認後再手動觸發 Step 3-5
- **手動模式**：David 說「今天的故事」，逐步執行

### 7.3 既有 Skill 的關聯

| 既有 Skill | 與本專案的關係 |
|------------|--------------|
| `art-in-teaching` | 可用於搜尋 Met/Europeana 的臺灣相關藝術品圖像 |
| `beautify` | 將故事輸出為精美 HTML 版本（投影用） |
| `teaching-log` | 記錄故事在課堂使用後的回饋 |
