# 「臺灣的故事」專案計畫書 v0.1

> **定位**：獨立第三條工作線，長期累積型教學素材生產系統
> **發起人**：David
> **日期**：2026-03-21
> **狀態**：草稿，待 David 審閱

---

## 一、專案願景

每天產出一篇「臺灣的故事」——精煉的文字、動人的畫面、有憑有據的史料——長期堆疊成一座教育的搖籃。學生在這個環境中持續沉浸，自然培養對臺灣土地的主體意識。

不是一次性的教案，而是一座慢慢長出來的故事庫。

---

## 二、核心設計原則

1. **有憑有據**：每篇故事必須錨定在至少一個可溯源的真實資料（檔案、照片、地圖、文物、口述）
2. **主題先行**：先有大區塊骨架，篇章在脈絡中生長，不是孤立的每日隨機
3. **課程對齊**：以一年為單位、四季為節奏，配合五年級臺灣史地主課程的教學進度
4. **三件套產出**：每篇故事固定產出文字內容 + 教師說書稿 + 投影圖像參考
5. **跨齡可擴展**：從五年級起步，未來可向上延伸至六到九年級（深度遞增）

---

## 三、主題區塊骨架（一年四季）

以五年級「臺灣史地」主課程為主軸，配合華德福第二個七年週期（7-14 歲）的學習特質——圖像、故事、情感連結為認知媒介。

### 秋季學期（9-11 月）——「這片土地的記憶」

**區塊 A：島嶼的誕生與最初的人（史前—原住民）**
- 臺灣的地質形成、山脈與海岸、河川、海洋、氣候
- 史前遺址（卑南、十三行、大坌坑）
- 南島語族的遷徙與定居
- 各族群的土地故事（泰雅的獵場、阿美的海、排灣的百步蛇）
- 預估篇數：15-20 篇

**區塊 B：海上來的人（大航海時代 1540s-1620s）**
- 葡萄牙人眼中的 Formosa
- 東亞海域的貿易網絡
- 臺灣在世界地圖上的出現
- 早期漢人與日本商人的足跡
- 預估篇數：8-12 篇

### 冬季學期（12-2 月）——「被書寫的島嶼」

**區塊 C：紅毛城的時代（荷蘭與西班牙 1624-1662）**
- 荷蘭東印度公司在大員（臺南）
- 西班牙人在雞籠（基隆）與淡水
- 荷蘭人與原住民的互動（新港文書）
- 鄭芝龍與海上勢力
- 預估篇數：12-15 篇

**區塊 D：國姓爺與他的後人（鄭氏時期 1662-1683）**
- 鄭成功的傳奇——英雄敘事適合十歲兒童的崇拜階段
- 從軍事佔領到農墾開發
- 鄭經與陳永華——制度建設
- 明鄭的終結與臺灣的轉折
- 預估篇數：10-12 篇

### 春季學期（3-5 月）——「帝國邊陲的日常」

**區塊 E：清帝國下的臺灣（1683-1895）**
- 渡海禁令與偷渡客
- 械鬥、開墾與水利（瑠公圳、曹公圳）
- 茶、樟腦、糖——臺灣的物產與世界
- 原漢關係與「番界」
- 林爽文事件、戴潮春事件——反叛與正義
- 劉銘傳的現代化——鐵路、電報、郵政
- 預估篇數：20-25 篇（最長區塊，內容最豐富）

### 夏季學期（6-7 月）——「新的開始」

**區塊 F：日本時代的臺灣（1895-1945）**
- 馬關條約與乙未戰爭
- 基礎建設——鐵路、公路、水庫、學校
- 霧社事件——莫那魯道
- 嘉南大圳與八田與一
- 日常生活——市場、學校、神社
- 預估篇數：15-20 篇

**區塊 G：我們的時代（1945— ）**
- 光復與二二八
- 白色恐怖與戒嚴
- 經濟奇蹟
- 民主化運動
- 預估篇數：10-15 篇（五年級點到為止，留給高年級深化）

**全年預估：90-120 篇**（以學期週數計算，扣除假期約 180 個上課日，每日一篇綽綽有餘，可保留彈性日做回顧與串接）

## 四、單篇故事產出模板

每篇故事固定產出五份文件，存入 `stories/[區塊]/[ID]/` 子資料夾：

### 4.1 故事正文（content.md）

含 YAML frontmatter（id、title、block、sub_theme、era、date）+ 故事本文 + 史實錨點 + 地理標記 + 延伸線索。字數以自然篇幅為準（實際產出約 650-1000 字）。五年級語言定位：具體、感官、有人物視角，不說教不灌輸。

### 4.2 教師說書稿（narration.md）

開場（1-2 分鐘）→ 說書本文（5-8 分鐘，含語氣標註、停頓標記、華德福吸氣-呼氣節奏）→ 圖像展示時機 → 收尾（1 分鐘，留懸念連結明天）→ 教師備註（學生提問預判、情感注意點、ABCD 差異化回應）。

### 4.3 投影圖像清單（images.md）

主圖 + 輔助圖（地圖/文物/建築），每張含檔案路徑、來源、授權、說明。附投影建議與無投影時的替代方案。

### 4.4 黑板畫 prompt（chalkboard-prompt.md）

供 Gemini 生成黑板畫風格圖像的中文 prompt。4 層搜尋策略：直接生成 → 風格調整 → 參考圖 → 手動描述。

### 4.5 原始素材包（raw-materials.md）

story-research 步驟的完整產出，含多資料源搜尋結果、AI 補充研究、交叉比對紀錄，作為故事撰寫的原始依據保留。

## 五、資料源盤點與可用性評估

### 第一層：有程式化 API，可自動搜尋

| 資料源 | 代碼 | API 狀態 | 需要 Key | 內容特色 |
|--------|------|----------|----------|----------|
| 國家文化記憶庫 TCMB | tcmb | 有 Swagger，待 Key 下來 | 是（已申請） | CC 授權文化素材，圖文豐富 |
| 政府資料開放平臺 | datagov | 有 REST API | 否 | 結構化資料集（古蹟、228 紀念碑等） |
| 文化資產國家網路 BOCH | boch | 端點回 404，疑似遷移 | 否 | 古蹟/歷史建築/考古遺址 |

**BOCH 狀態待確認**：`data.boch.gov.tw/opendata/v2/` 目前回 404，可能已遷移至 data.gov.tw 或新網域。需手動查證最新 API 文件。

### 第二層：有網站可瀏覽，需 AI 輔助研究

| 資料源 | 代碼 | 網站狀態 | 內容特色 |
|--------|------|----------|----------|
| 國家檔案資訊網 AA | archives | 運作中 | 228/白色恐怖政治檔案，數位影像可線上瀏覽 |
| 臺灣記憶 TM | tm | 運作中 | 老照片、碑碣、古書契、日治明信片、地方志 |
| 開放博物館 OpenMuseum | openmuseum | 運作中 | 中研院跨館策展，社群貢獻內容 |

| 資料源 | 代碼 | 狀態 | 備註 |
|--------|------|------|------|

### 補充資料源（建議新增）

- **維基共享資源**：大量 CC 授權臺灣歷史圖像

## 六、每日生產管線設計

### 6.1 管線流程（story-daily v2.2.0，8 步驟 + Checkpoint 矩陣）

```
[每日觸發] Scheduled Task（Mon-Fri 10:30）或手動「今天的故事」
    │
    ├── Step 1: 選題（story-planner skill）【需教師確認，排程模式預授權】
    │   ├── 讀取 theme-skeleton.yaml → 確認當前區塊與子主題
    │   ├── 讀取 index.yaml → 避免重複
    │   └── 產出：今日故事主題 + 搜尋關鍵字
    │   └── ✓ Checkpoint 1：主題已確認、關鍵字已產出
    │
    ├── Step 2: 資料搜尋（story-research skill）
    │   ├── 呼叫 taiwan_history_api.py 搜尋多資料源
    │   ├── AI 補充研究（網路搜尋、交叉比對）
    │   └── 產出：raw-materials.md（文字 + 圖像連結 + 來源標註）
    │   └── ✓ Checkpoint 2：素材包 ≥ 3 來源、含圖像連結
    │
    ├── Step 3: 故事撰寫（story-writer skill）
    │   ├── 依模板產出五件套（content + narration + images + chalkboard-prompt + raw-materials）
    │   ├── 事實查核：每個史實錨點必須有來源
    │   └── 產出：五份 .md 文件，存入 stories/[區塊]/[ID]/
    │   └── ✓ Checkpoint 3：五檔齊全、frontmatter 完整
    │
    ├── Step 4: 品質檢查（story-verify skill）【可與 Step 5 平行】
    │   ├── 多維度檢核：事實查核、敘事品質、說書稿、圖像清單
    │   ├── FAIL / WARN 分級判定
    │   └── 產出：品質報告（reviews/[ID]-quality.md）
    │   └── ✓ Checkpoint 4：FAIL = 0
    │
    ├── Step 5: Gemini 圖像生成【可與 Step 4 平行】
    │   ├── 開啟 Google Gemini → 以中文前綴觸發生成
    │   ├── 黑板畫 prompt → 生成黑板畫風格圖像
    │   └── 產出：黑板畫 PNG（~/Downloads/）
    │   └── ✓ Checkpoint 5（FAIL）：互動模式必須嵌入黑板畫
    │
    ├── Step 6: 組裝美化（assemble-story.js + beautify）
    │   ├── Markdown → 精美 HTML（華德福四季視覺模板）
    │   ├── HTML → PDF（Puppeteer，A4 固定版面）
    │   ├── 嵌入黑板畫、pullQuote、下篇預告（動態提取）
    │   └── 產出：HTML + PDF（temp/beautify-[ID]-完整版.*）
    │   └── ✓ Checkpoint 6：HTML + PDF 輸出、檔案大小 > 0
    │
    ├── Step 7: 上傳 Google Drive（gws upsert）
    │   ├── 先刪舊版再傳新版（每篇只保留最新 HTML+PDF）
    │   └── 產出：Google Drive 連結
    │   └── ✓ Checkpoint 7：上傳成功確認
    │
    └── Step 8: 歸檔與索引更新（story-archive skill）
        ├── 更新 index.yaml（含標籤、地理、時間軸）
        ├── 更新 session.yaml pipeline_outputs
        └── 產出：更新後的 index.yaml + session.yaml
        └── ✓ Checkpoint 8：索引已更新、session 已同步
```

**Checkpoint 矩陣規則**：每步轉換點強制驗證，FAIL 立即中止，WARN 允許繼續但記錄。排程模式除 Step 1 外全部預授權。
