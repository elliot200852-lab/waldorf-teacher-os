# Voice Calibration Log

## 使用方式

1. **寫作前**：讀取本檔案，注意最近 10 筆回饋，特別是與當次類別相同的條目
2. **寫作後**：若 David 標記了「不像我」或給了修改意見，AI 必須新增一筆回饋記錄
3. **定期萃取**：當同一類別累積 >= 5 筆 pending 回饋時，AI 應提議將反復出現的模式萃取回對應的 voice-samples/ 校準筆記中，並將狀態改為 extracted

## 回饋記錄格式

每筆回饋按以下格式記錄，按時間倒序排列（最新的在最上面）：

```
### [YYYY-MM-DD] [類別編號+名稱] [一句話摘要]

- **原文（AI 寫的）**：「...」
- **David 的修改 / 評語**：「...」
- **萃取的規則**：（一句話概括，日後可回寫到校準筆記）
- **狀態**：pending / extracted
```
