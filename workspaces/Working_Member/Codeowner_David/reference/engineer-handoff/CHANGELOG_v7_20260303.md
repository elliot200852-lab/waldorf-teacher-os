## 變更清單

### 一、Git 多人協作模型

- 選定 **Collaborator + Branch** 模式（非 Fork 模式），確保教師能互相看到工作
- Branch 命名規則：`workspace/{Teacher_ID}`（例如 `workspace/Teacher_林信宏`）
- 三層保護機制：Pre-commit Hook（軟）→ CODEOWNERS（硬）→ Branch Protection（硬）
- Branch Protection 只保護 main 的合併，教師在個人 branch 上自由 commit

### 二、新增技能：save（存檔）

- **觸發語**：「存檔」「儲存」「幫我存」「commit」「上傳」「備份今天的工作」
- **檔案**：`ai-core/skills/save.md` + `.claude/commands/save.md`
- **流程**：檢查變更 → 取得說明 → 執行 git add/commit/push → 確認結果
- **容錯**：push 失敗時自動嘗試 `git pull --rebase`

### 三、新增技能：pull-request（合併申請）

- **觸發語**：「發 PR」「合併申請」「送回主系統」「通知 David 合併」
- **檔案**：`ai-core/skills/pull-request.md` + `.claude/commands/pull-request.md`
- **流程**：檢查未存檔 → 收集 PR 資訊 → 確認 branch → 引導 GitHub PR 建立 → 選用 gh CLI
- **安全**：PR base 永遠是 main，只有 David 可合併

### 四、新增腳本：save.sh（一行存檔）

- **路徑**：`setup/save.sh`
- **用法**：`bash setup/save.sh "完成 Unit 2 教案"`
- **功能**：自動執行 git add → commit → push，帶彩色輸出

### 五、add-teacher.sh 升級

- **新增 Step 6**：`create_teacher_branch()` — 建立教師個人 branch 並推送至 GitHub
- **print_summary 升級為 Step 7**：顯示 branch 名稱

### 六、quick-start.sh 升級

- **新增 Check 8**：「切換到你的個人工作分支」
  - 偵測遠端 `workspace/` 開頭的 branch
  - 單一 branch → 自動 checkout
  - 多個 branch → 列出選項
  - 無 branch → 提示聯絡 David
- 結尾確認是否已在 workspace branch 上

### 七、teacher-guide.md 重寫

  - Part 3：學期末合併申請（AI 口語 + GitHub 網頁）
- **第十四節更新**：快速參考表改為 AI 口語指令為主
