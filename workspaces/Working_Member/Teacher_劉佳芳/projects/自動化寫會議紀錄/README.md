# 自動化寫會議紀錄專案

這裡是用來開發與測試「自動化會議紀錄」功能的資料夾。

## 目錄結構簡介
- `src/`: 放置將用來處理會議資料的程式碼 / 腳本。
- `templates/`: 放置會議紀錄的樣板檔案（例如預設的 Markdown 排版格式）。
- `data/raw_notes/`: 用來放置測試用的**原始會議草稿**或零碎記錄（Input）。
- `data/output/`: 用來放置程式自動生成的**最終會議紀錄檔案**（Output）。

## 測試流程說明
日後開發若有進展，我們會在此資料夾進行測試。您可以將您的原始紀錄放進 `data/raw_notes/`，我們寫好的程式會將轉譯結果輸出在 `data/output/` 供您檢視。

## 執行原則
1. **來源為準**：只處理來源 Google Drive 資料夾（`1T7oIUVK21BE8xBgo788TpRyYTTt9wxOS`）中實際存在的錄音檔，不在此資料夾中的一律跳過。
2. **內容驗核**：若 Gemini 判斷錄音內容並非會議性質（例如戲劇、廣播劇等），該檔案視為不符規則，自動跳過，不產出會議紀錄。
3. **上傳後清除**：MD 會議紀錄檔案成功上傳至雲端資料夾後，立即刪除 `data/output/` 中的本地 MD 檔，不留存本地副本。雲端為唯一保存位置。
