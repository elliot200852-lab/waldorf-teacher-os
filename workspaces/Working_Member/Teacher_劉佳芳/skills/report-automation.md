# 實驗教育計畫結案報告自動化技能

此技能可協助教師自動從 Google Drive 提取指定學年度的參考格式、年度大計畫以及各項研習資料，並彙整為一份結構嚴謹、符合正式語氣的結案報告。

## 執行工作流 (Execution Workflow)

### Phase 1: 初始化與核心資料讀取
1. **讀取參考格式**：
   - 導航至「歷年資料」路徑下的最新結案報告，分析其目錄結構與語氣。
   - 預設參考：`14_oBbOgsVGLoBemO2RabhNJfDW09Qon1` (113學年度成果報告)。
2. **讀取該學年度大計畫**：
   - 讀取該學年度的「實驗教育計畫草案」。
   - 提取：核心宗旨、總體目標與預期效益。

### Phase 2: 研習子報告批次處理
1. **遍歷研習資料夾**：進入對應年度的「研習紀錄」資料夾（如 `1_rWOMWUxQVA9LyBO1Ehf2sUHbIGbOVjK`）。
2. **提取與撰寫**：針對每個研習子資料夾，讀取「課程計畫」與「教師回饋」。
3. **子報告生成**：
   - **課程介紹**：將條列式目標轉化為流暢敘述。
   - **課程回饋**：彙整滿意度與建議，以正向客觀語氣撰寫成效。
   - **註記**：若缺漏資料則標註「資料從缺」。

### Phase 3: 總報告綜合整併
1. **套用骨架**：以 Phase 1 的結構為基礎。
2. **融合內容**：填入年度摘要、各項研習成果。
3. **優化語言**：統一為「教育行政正式用語」，避免第一人稱。

### Phase 4: 匯出與存檔
1. **生成 Google 文件**：使用 GWS 指令建立雲端文件。
2. **同步 MD 文件**：上傳 Markdown 格式備份至目標資料夾（方便快速預覽與二次轉換）。
